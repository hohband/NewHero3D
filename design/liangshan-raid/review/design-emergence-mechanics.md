# 「劫寨」涌现性增强机制设计

> 日期：2026-07-20
> 基准文档：combat-mode-design.md + 4 GDD + 涌现性审查 + 好玩性分析
> 设计原则：每个新机制必须满足两条——①能与至少 2 个现有系统产生**非预设交互**；②玩家可**观察到、学习到、利用到**这种交互，而非纯后台规则。

---

## 0. 设计总纲：什么是"涌现式机制"

一个机制是涌现式的，当且仅当：

> **该机制引入的不是一个新答案，而是一个新变量。**
> 变量 × 现有变量 → 组合爆炸，而非叠加。

反例（拼图式）：L4 法术塔"驱散召唤" → 玩家的答案从"用召唤"变成"不用召唤"。只是换了答案。

正例（涌现式）：L4 引入"锋火台"——被玩家占领后可释放假信号。玩家选择：占领锋火台骗走一波援军（省力但花时间）vs 直接硬刚（快但风险大）。锋火台 × 时迁潜行 = 静默占领；锋火台 × 吴用援军 = 假扮援军迷惑敌军……答案不是设计师给的。

## 1. 信息战系统（Information Warfare）

> **核心涌现原理**：引入"不完全信息" → 玩家必须在不确定中做决策 → 不同玩家用不同的信息收集策略 → 同一关卡在不同信息策略下完全不同。

### 1.1 战争迷雾（Fog of War）

**机制定义**

战场初始被迷雾覆盖。玩家只能看到：
- 己方单位的 **视野范围**（默认 4 格，远程单位 5 格，时迁 6 格）
- 侦查期（8s）期间**全图短暂可见**，侦查期结束后迷雾恢复
- 侦查期标记过的建筑/陷阱**持续可见**（坐标已知）

迷雾中敌人**真实存在且持续行动**，只是玩家看不见。

**涌现点**

| 涌现交互 | 说明 |
|---------|------|
| 迷雾 × 侦查期 | 8s 侦查太短，玩家必须选择"重点侦察哪片区域"——是看外墙结构还是找陷阱分布？不同选择导致完全不同的初始信息态 |
| 迷雾 × 时迁 | 时迁高视野+潜行=天然侦察兵。玩家可以"先放时迁探路→根据发现调整后续部署"。时迁从"奇袭核心"扩展为"情报核心" |
| 迷雾 × 吴用援军 | 援军虽然弱，但可以当肉侦——故意扔进迷雾触发敌人暴露位置 |
| 迷雾 × 远程单位 | 花荣超远射程但需要视野→需要其他单位提供视野才能发挥最大价值。制造"前哨+狙击"的涌现式兵种协同 |
| 迷雾 × 巡逻队（见 §1.3） | 巡逻队在迷雾中移动→听到脚步声但看不到具体位置→"那边有动静"的紧张感 |
| 迷雾 × 模板随机偏移 | 迷雾让"每局布局不同"从视觉变化升级为策略变化——你不知道墙在哪、塔在哪，必须实地侦察 |

**实现建议**

- 视野计算在逻辑层（`visibility_system`），每个单位维护一个 `visible_tiles` 集合
- 表现层用 shader 或 tile 覆盖渲染迷雾
- 侦查期标记物（玩家单击建筑→记录坐标→迷雾恢复后该建筑保持半透明显示）
- 哨兵和巡逻队的"脚步声"在迷雾中以方向性音频提示（只听声、不见人）

### 1.2 哨兵（Sentries）

**机制定义**

哨兵是**固定位置的敌方观察单位**，核心行为链：

```
哨兵发现敌方 → 点烽火（2s 动作）→ 烽火升起（全图可见光柱）
                    ↓
              提升警报等级 +1（见 §2.1）
                    ↓
              周围巡逻队向哨兵位置集结
```

哨兵属性：

| 属性 | 值 | 说明 |
|------|-----|------|
| 视野 | 5 格（扇形 120°，非 360°） | 有盲区，可从背后接近 |
| HP | 极低 | 1-2 次攻击可击杀 |
| 烽火点燃时间 | 2s | 窗口期——"在他点火前干掉他！" |
| 分布 | 外墙门口 / 塔旁 / 核心入口 | 关键位置 |

**涌现点**

| 涌现交互 | 说明 |
|---------|------|
| 哨兵扇形视野 × 巡逻队路线 | 玩家需要计算"巡逻队走远→哨兵视线转过去→我从背后潜入"的 timing 连环谜题，而这完全是位置关系涌现的，不是脚本 |
| 哨兵 × 花荣 | 花荣的百步穿杨冷却 12s——如果两个哨兵距离近，能不能一箭双杀？如果能→涌现式技巧；如果不能→必须分配击杀优先级 |
| 哨兵 × 时迁 | 时迁妙手可以**静默暗杀**哨兵（不触发烽火）。时迁从"拆陷阱"扩展为"暗杀哨兵"，战场角色丰满了 |
| 哨兵 × 李逵 | 李逵破墙进场的巨响会**立刻吸引哨兵转身**。所以破墙点必须选在哨兵视线外 |
| 哨兵 × 烽火连锁 | 一个哨兵点燃烽火→相邻哨兵自动转向烽火方向→它们的背面暴露→创造出"用烽火吸引注意、从后面潜入"的战术窗口。这是纯机制涌现，不需脚本 |
| 哨兵 × 战争迷雾 | 哨兵在迷雾中是一个"移动的光点"（手中提灯）——玩家在迷雾中看到光点就知道那里有哨兵，但不知道哨兵面向哪边 |

**实现要点**

- 哨兵的视野扇形在逻辑层计算（角度+距离），表现层画半透明锥形
- 烽火升起后作为地图上的 persistent 光源，所有单位可见
- 哨兵有"警觉"和"正常"两种状态：听到附近脚步声→警觉（转身面向声源）→2s 内无进一步异常→恢复正常

### 1.3 巡逻队（Patrol Squads）

**机制定义**

巡逻队 = 2-3 名庄丁沿**随机偏移路线**循环移动。每局路线不同（同关卡模板、不同偏移）。

| 属性 | 值 | 说明 |
|------|-----|------|
| 规模 | 2-3 人 | 小型编队 |
| 视野 | 3 格（360°） | 比哨兵近但全方位 |
| 移速 | 慢（巡逻速度）→ 快（追击速度） | 发现敌人后加速 |
| 行为 | 巡逻 → 发现敌人 → **不立即攻击，先跑回报警** | 核心：巡逻队是"信息节点"不是"战斗单位" |
| 路线 | 模板路线 + 随机偏移（±2 格） | 每局不同 |

**行为链（关键设计）：**

```
巡逻队发现敌人
    → 追击速度冲向最近哨兵/塔（不攻击敌人）
    → 若中途被击杀 → 警报不触发
    → 若成功到达哨兵/塔 → 警报等级 +1，全体巡逻队朝该区域集结
    → 若所有哨兵已死 → 巡逻队转为直接战斗
```

**涌现点**

| 涌现交互 | 说明 |
|---------|------|
| 巡逻队"跑回报警" × 林冲突进 | 巡逻队看到你→转身就跑→林冲必须突进追上截杀。制造"追与逃"的紧张时刻。林冲的突进技能此时不是"切入核心"而是"追杀信使"——同一技能在不同情境下的涌现式用法 |
| 巡逻队 × 多部署点（L3+） | 玩家可以在 A 点放肉盾吸引巡逻队→巡逻队跑向 A→B 点偷渡。声东击西不是脚本，是巡逻队 AI 的涌现结果 |
| 巡逻队 × 鲁智深嘲讽 | "嘲讽"对巡逻队是否有效？如果有效→鲁智深可以硬留住巡逻队不让它报警。如果无效（巡逻队优先跑）→需要花荣远程截杀。两种设计都产生涌现 |
| 巡逻队 × 时间压力 | 巡逻队走一圈约 30-45s。玩家需要：推算巡逻队现在在哪→计算我还有多少秒安全窗口→决定"现在突击"还是"等它走过去"。时间估算能力成为玩家技能 |
| 巡逻队 × 迷雾 × 哨兵信号 | 迷雾中看到远处烽火亮起→知道"那边有人被发现了"→但这是敌人发现了我的单位，还是另一条巡逻队误报了？信息的不完整性本身就是涌现 |

**设计决策建议**：巡逻队受鲁智深嘲讽影响——这让鲁智深的技能产生新用法（"嘲讽拉住巡逻队，队友趁机通过"），比"免疫嘲讽"产生更多涌现。

### 1.4 视野博弈总表

| 己方 | 敌方 | 交互 |
|------|------|------|
| 单位视野（4-6格） | 哨兵扇形视野（5格/120°） | 玩家可绕扇形盲区 |
| 时迁潜行（不被索敌） | 哨兵听觉（听到脚步声→警觉） | 潜行≠隐身，时迁也要绕路 |
| 花荣超远程（7格射程） | 瞭望塔射程（7格） | 花荣可在塔射程边缘狙击哨兵，但需要前线单位提供视野 |
| 侦查期 8s 全图 | 迷雾恢复后未知 | 侦查期标记决定整局信息优势 |
| 烽火信号（全图可见） | 巡逻队响应 | 一个烽火可能打草惊蛇，也可能用来声东击西 |

## 2. 动态防御系统（Dynamic Defense）

> **核心涌现原理**：防御方从"固定谜题"变成"反应式对手" → 玩家的每个行为都会改变敌方状态 → 敌方状态变化反过来改变玩家的最优策略 → 形成博弈循环，而非单向解题。

### 2.1 警报等级系统（Alert Level）

**机制定义**

敌方基地有一个 0-4 级的警报状态。等级越高，防御越激进。

| 等级 | 名称 | 触发条件 | 敌方行为变化 | 玩家策略含义 |
|------|------|---------|-------------|-------------|
| **0** | 平静 | 开局默认 | 哨兵正常巡逻、庄丁在营房待命（不在战斗位）、塔射程-1、陷阱未激活 | "黄金窗口"——潜行/速攻的最佳时机 |
| **1** | 可疑 | 哨兵听到脚步声（但未确认敌人） | 哨兵警觉（转身面向声源）、巡逻队移速+30%、瞭望塔开始旋转搜索 | "还能潜行，但更难了"——需要更精确的 timing |
| **2** | 警戒 | 哨兵点燃烽火 / 巡逻队成功报警 / 任何敌方单位被击杀 | 庄丁从营房出动占领战斗位、所有塔射程恢复满、巡逻队路线改为"搜索模式"（扩大巡视范围）、**援军开始从地图外分批抵达** | "潜行窗口关闭"——转为正面战斗。但如果你已经潜入核心附近…… |
| **3** | 激战 | 外墙被破 / Boss 守将 HP < 100% / 庄丁伤亡 ≥ 30% | Boss 守将从核心出击、法术塔激活（若有关卡机制）、敌方指挥官开始使用技能（见 §2.2）、所有陷阱自动激活 | "全面开战"——需要 combo 链和战时指令 |
| **4** | 绝望 | 核心 HP < 30% | 敌方发动"焦土战术"：庄丁获得狂暴（攻速+50%但持续自伤）、部分建筑开始自燃制造火墙、Boss 守将进入不可打断的终极技能 | "最后一搏"——要么速杀核心，要么被反扑淹没 |

**关键设计**：警报等级只升不降。玩家**无法"消警报"**，只能选"在低等级时做多少事"。

**涌现点**

| 涌现交互 | 说明 |
|---------|------|
| 警报等级 × 编队选择 | "全带潜行速攻流"（时迁+林冲+燕青）追求在等级 0-1 结束战斗；"全带正面流"（鲁智深+李逵+公孙胜）不在乎警报等级。两种编队在同一个关卡中是**两款不同的游戏** |
| 警报等级 × 劫掠决策 | 警报 0 时劫掠资源建筑很容易→但劫掠会提高警报！玩家面临："先劫掠（提升警报、更难打核心）还是先打核心（放弃劫掠收益、换更简单的 Boss 战）？" |
| 警报等级 × 多部署点 | L3+ 多部署点：A 点佯攻（触发警报 2，吸引援军）→ B 点主力趁援军被吸引走时突入。佯攻是真实的战斗（不是脚本！），因为警报系统真实存在，敌方的反应也是真实的 |
| 警报等级 × 援军系统 | 援军不是"定时刷新的怪"，而是"警报等级的产物"。如果玩家全程保持警报 0，不会出现援军。玩家可以"以慢打快"全程潜行避免援军→或者"以快打快"在援军抵达前结束 |
| 警报等级 × Boss 行为 | Boss 在警报 3 时才出击。如果玩家在警报 0-2 时已经潜入核心区域→Boss 一出来面对的是已经摆好阵型的梁山好汉 |

**设计中需要特别注意的**：确保"全程警报 0 通关"在理论上是可能的（某些关卡、某些编队），但不能是唯一最优解——正面战斗（警报 3-4）应该有对应的收益（如劫掠更多、用时更短、额外击杀奖励）。

### 2.2 敌方指挥官 AI（Commander AI）

**机制定义**

每个关卡有一个"指挥官"（L1 祝氏三兄弟轮值、L2 史文恭、L3 梁中书……），它不直接战斗，而是：

1. **每 15-20s 做一次决策**（模拟指挥周期）
2. **决策基于它"知道"的信息**（受限于敌方视野，不是上帝视角）
3. **消耗"指挥点数"（Command Points, CP）**来执行行动

| 指挥官行动 | CP 消耗 | 效果 |
|-----------|---------|------|
| 调动巡逻队 | 1 | 将一队巡逻队路线改为指定区域 |
| 增援某区域 | 2 | 在指定塔/门召唤 3 名庄丁增援 |
| 激活陷阱 | 1 | 指定区域的陷阱从"被动触发"变为"自动触发" |
| 塔集火指令 | 2 | 指定一座塔优先攻击指定目标（无视索敌优先级） |
| 关闭闸门 | 3 | 一道内墙的闸门关闭，阻挡通行（可被破坏，HP 极高） |
| 释放法术（仅 L4+） | 3 | 驱散指定区域内的所有己方增益 |

指挥官每 15s 恢复 1 CP，上限 5 CP。这意味着指挥官的决策是**有限资源下的取舍**。

**涌现点**

| 涌现交互 | 说明 |
|---------|------|
| 指挥官"不知道"的事 × 战争迷雾 | 如果玩家始终没被哨兵/巡逻队发现→指挥官不知道玩家在哪、有多少人→它只能用 CP 做"常规巡逻"→浪费 CP。信息优势产生了真实的战术优势 |
| 指挥官"被骗" × 佯攻 | 玩家在 A 墙用 2 个援军+公孙胜 AOE 制造大动静→指挥官认为主攻方向是 A→把所有 CP 花在 A 路增援→B 路空虚。佯攻不是"假装"，是真的在打——指挥官的反应也是真的 |
| 指挥官决策循环 × 玩家节奏 | 指挥官每 15-20s 决策一次。玩家掌握了这个节奏后，可以"在指挥官决策窗口前制造假象、决策窗口后立刻变招"。这创造了一个 meta 层面的 timing 博弈 |
| 指挥官 × 斩杀优先级 | 如果玩家优先击杀史文恭（Boss+指挥官二合一）→敌方失去指挥→不再有 CP 决策→所有庄丁回到预设 AI。斩首行动有双倍收益 |
| 指挥官 × 英雄专长 | 不同关卡指挥官有不同的**决策偏好**。祝氏三兄弟"偏好增援外墙"、史文恭"偏好集火远程单位"、高廉"偏好用法术驱散"。玩家需要了解指挥官的性格——这和原著的性格设定对接 |

**实现要点**

- 指挥官 AI 的核心是一个 **效用评估函数**：对每个可能的行动计算效用值（威胁度 × 可执行性 × 资源充足度），选最高效用的行动
- 指挥官的信息状态维护一个"已知敌人位置"集合，每次哨兵/巡逻队/塔发现敌人时更新
- 不同指挥官有不同的权重参数（偏好），从 CSV 读取

### 2.3 庄丁兵种分化

**机制定义**

取代当前单一的"庄丁"单位，分化为 6 种兵种，每种有不同的 AI 行为。

| 兵种 | 定位 | HP | 移速 | 行为 AI | 涌现交互 |
|------|------|-----|------|---------|---------|
| **枪兵** | 前排 | 中 | 慢 | 列阵于墙后/门前，**反突进**：对冲锋/位移中的单位伤害 ×2 | 林冲突进必须等鲁智深先嘲讽拉开枪兵阵型 |
| **弓手** | 远程 | 低 | 中 | 优先射击**低 HP 单位**和**正在施法的单位** | 公孙胜施法时最脆弱→需要鲁智深保护 |
| **哨探** | 信息 | 极低 | 极快 | 发现敌人后**立即逃跑**，跑向最近哨兵 | 花荣的"优先锁定"需要判断打哨探还是打弓手 |
| **力士** | 肉盾 | 高 | 慢 | 守卫核心/资源建筑门口，**不追击** | 时迁可以绕过力士（背后无视野）但不能正面通过 |
| **火攻手** | 特殊 | 中 | 慢 | 投掷火油罐，制造**持续燃烧地面**（每秒伤害+减速） | 燃烧地面会被雨水熄灭（§3.1）→天气改变兵种价值 |
| **医师** | 辅助 | 低 | 中 | 给附近受伤庄丁**回血**，优先治疗 Boss | 玩家必须优先切后排医师，否则 Boss 战无限拉长 |

**涌现点**

兵种分化的核心涌现来自**兵种 × 英雄的石头剪刀布关系**：

```
枪兵 → 克制 → 林冲（突进被反制）
  ∧              ∨
  │           鲁智深（嘲讽破阵）
  │              ∨
  │           李逵（AOE 全清）
  └──────────────┘
```

但这不是硬编码的克制关系——而是 AI 行为自然产生的。枪兵的 AI 是"列阵反突进"→林冲突进吃亏→但林冲可以从侧面突进（枪兵只防正面）→这是玩家可以学习的涌现性技巧。

## 3. 环境系统（Environmental Systems）

> **核心涌现原理**：环境不是"背景"，而是"所有单位共享的全局状态变量"。环境改变→所有单位的相对强度改变→编队/战术的价值重新排序。

### 3.1 天气系统

**机制定义**

开局随机天气（或特定关卡固定天气）。天气影响全局。

| 天气 | 视觉效果 | 机制效果 | 谁受益 | 谁受损 |
|------|---------|---------|--------|--------|
| ☀️ **晴** | 默认 | 无 | — | — |
| 🌧️ **雨** | 屏幕雨丝、地面湿润 | 火系效果无效、所有单位移速 -10%、视野 -1 格、公孙胜雷法伤害 ×1.5（导电） | 公孙胜（雷法增强）、林冲（突进不受移速影响） | 火攻手（火油无效）、弓手（视野受限） |
| 🌫️ **雾** | 浓雾粒子、远景模糊 | 视野 -3 格、远程射程 -2、潜行单位更难被发现（+10% 闪避） | 时迁（更难被发现）、李逵（近战不受影响） | 花荣（射程废了一半）、瞭望塔（侦察范围大减） |
| 💨 **风** | 树叶飘动、旗帜方向 | 顺风方向移速 +20%、逆风移速 -20%、火焰/烟雾随风扩散 | 顺风方向进攻方获得移速优势 | 逆风方向寸步难行 |
| ❄️ **雪** | 地面白色、雪花飘落 | 移速 -20%、单位留下**足迹**（持续 8s，暴露潜行轨迹） | 花荣（远程=不受移速影响）、肉盾（本来就慢） | 时迁（足迹暴露潜行！必须绕路或等足迹消失） |

**涌现点**

天气系统是**最低成本、最高涌现产量**的设计投资。核心在于：天气改变了每个英雄的价值排序，而玩家必须当场调整编队和战术。

| 涌现交互 | 说明 |
|---------|------|
| 雨天 × 编队决策 | "这局是雨天，公孙胜雷法 +50%，那我选他当核心→配合鲁智深聚怪→林冲补单体。燕青和花荣这局暂时放了（远程被视野削弱）。"同一个关卡，雨天和晴天的编队完全不同 |
| 雾天 × 潜行流 | 雾天时迁+林冲潜行流是 S 级战术，因为敌人视野同时被雾削弱。但雾天也可能遇到意外——你撞上了同样看不清的巡逻队！ |
| 雪天 × 时迁 | 时迁的足迹暴露了他的路径。敌人会沿着足迹反向追踪！玩家必须让时迁走迂回路线、或者利用暴风雪间歇期（足迹被新雪覆盖）。追踪与反追踪变成玩法 |
| 风 × 火攻 | 顺风放火→火势蔓延到敌方建筑；逆风放火→烧回自己。玩家需要读风向决定是否使用火攻→甚至用公孙胜技能改变天气（消耗大量粮草） |
| 天气 × 粮草经济 | 公孙胜可以消耗大量粮草施法**暂时改变天气**（如祈雨灭火）→这让粮草有了一个全新的用途：战略性战场改造 |

### 3.2 昼夜系统

**机制定义**

开局随机时段，或玩家可选。

| 时段 | 视觉效果 | 机制效果 |
|------|---------|---------|
| 🌅 **黎明** | 暖色光、长阴影 | 哨兵视野短暂 -1（日光刺眼），开局 20s 后恢复正常 |
| ☀️ **白昼** | 默认光照 | 无修正（基准） |
| 🌆 **黄昏** | 橙色光、阴影拉长 | 塔射程 -1（光线不足）、瞭望塔搜索速度减半 |
| 🌙 **黑夜** | 暗色光、单位自带光源 | 视野 -4（所有单位），但带有"夜战"标签的单位不受影响 |

**梁山好汉的"夜战"标签：** 李逵（天生夜战）、时迁（偷鸡摸狗专业户）、燕青（相扑手黑暗感知）。

**涌现点**

| 涌现交互 | 说明 |
|---------|------|
| 黑夜 × 编队 | "这局是黑夜，李逵+时迁+燕青天然优势→全近战夜战队。花荣废了。" 编队选择变成"看天吃饭" |
| 黑夜 × 敌方 | 庄丁也在黑夜中视野-4，但会**点燃更多火把**——火把是光源，暴露他们自己的位置。玩家的视野虽小，但能看到远处的火把光点 |
| 黑夜 × 烽火 | 烽火在黑夜中亮度翻倍——但同时暴露烽火周围所有敌军位置。烽火从"警报"变成了"双向信息源" |
| 日夜 × 天气叠加 | 黑夜+雾天 = 几乎全盲的恐怖体验，只有时迁和李逵能打 | 白昼+晴天 = 经典的正面对决 |

### 3.3 可交互战场物体

**机制定义**

散布在地图上的物体，玩家和敌方都可以利用。

| 物体 | 分布 | 玩家交互 | 敌方交互 | 涌现层 |
|------|------|---------|---------|--------|
| 🛢️ **火药桶** | 墙根、塔旁、仓库 | 远程射击引爆（范围爆炸，破坏相邻墙壁、伤害周围单位） | 庄丁在警报 3+ 时会**故意引爆火药桶**制造火墙阻挡玩家 | 玩家可以"抢在敌人引爆前射击火药桶"→时间竞赛 |
| 🍺 **酒坛** | 营房旁、核心附近 | 击碎→地面醉酒（减速 50%，持续 8s）；可被火源点燃 | — | 引诱敌人走进酒渍→点燃→连锁伤害 |
| 🌾 **粮草堆** | 资源建筑旁 | 劫掠（拿资源）或点燃（制造火墙阻挡追兵） | — | 同一个物体有两种互斥用法：拿资源 vs 造障碍 |
| 🪨 **滚石/檑木** | 山坡上 | 需要 3 名单位合力推动（靠近后出现"推动"按钮）→顺坡滚下碾压路径上所有单位 | 守军也可推动（警报 3+） | "抢占制高点"成为战术目标 |
| 🔥 **信号烽火台** | 城墙高处 | 占领后：①释放假烽火信号→迷惑敌方指挥官（下一波援军走错方向）；②点燃真烽火→吸引注意力（声东击西） | 哨兵优先守卫烽火台 | 时迁潜行占领烽火台→释放假信号→援军去了反方向→主力轻松破核心。一套完整的涌现式策略链 |
| 🌉 **吊桥** | 连接内外墙的通道 | 破坏绳索→吊桥坠落→路径切断 | — | 破坏吊桥可以阻断敌方援军路径，但也阻断了自己的退路 |

**涌现设计原则**：每个物体至少提供两种互斥用法，让玩家做选择而非"正确答案"。

### 3.4 环境连锁反应

多种环境元素之间的交叉涌现：

| 连锁 | 机制 | 玩家利用 |
|------|------|---------|
| 火药桶爆炸 → 引燃相邻粮草堆 → 粮草堆燃烧制造火墙 → 火墙阻挡援军路线 | 系统间自动产生，不需脚本 | 玩家可以刻意策划连锁：先不引爆火药桶，等援军走进范围再引爆 |
| 雨天 → 火系无效 → 火药桶不爆炸 → 但可以物理推动利用 | 天气覆盖所有物体 | 雨天时火药桶从"爆炸道具"变成"障碍物" |
| 风 → 火墙蔓延方向改变 → 可能烧回己方 | 风覆盖所有火源 | 玩家必须看风向决定放火位置 |
| 滚石碾压 → 经过酒渍区域 → 酒渍被推平但滚石带上酒味（无实际效果，纯趣味） | 物理 × 环境 | 这种无用的交互本身就是乐趣 |

## 4. 战场社会系统（Battlefield Social Systems）

> **核心涌现原理**：战场上不只有敌我双方，还有第三方利益相关者。第三方引入后，简单的"敌/我"二元关系变成"敌/我/中立"三角关系，三角关系天然产生涌现。

### 4.1 平民与内应

**机制定义**

地图上散布**平民**（避难村民、被关押的义士、梁山内应）。

| 类型 | 外观 | 行为 | 交互方式 |
|------|------|------|---------|
| 🏠 **避难村民** | 老人/妇女/儿童 NPC | 恐惧乱跑，战斗靠近时会尖叫逃跑（吸引守军注意） | 部署单位靠近会形成"保护区"→村民不再乱跑；战斗后结算：保护村民+声望 |
| 🔗 **被关押义士** | 牢笼中 | 不动，等待解救 | 摧毁牢笼→该义士成为**临时友方单位**（不占部署槽、不耗兵符），战斗后有机会永久招募 |
| 🕵️ **梁山内应** | 伪装成庄丁或商人 | 标记有梁山暗号（侦查期才能看到） | 时迁与内应交互→获得情报：揭示隐藏陷阱、标记 Boss 弱点、告知巡逻队路线 |

**涌现点**

| 涌现交互 | 说明 |
|---------|------|
| 平民 × 潜行流 | 如果走潜行路线→不会吓到平民→平民不尖叫→不暴露位置。但如果走正面路线→平民尖叫→暴露位置→更多敌人涌来。平民不是"背景"，是"潜行路线的奖励机制" |
| 平民 × 劫掠 × P4 支柱 | 劫掠粮仓时会吓到附近平民→扣除声望。但如果你先保护平民→再劫掠→不扣声望。同一个劫掠行为，在不同操作顺序下产生不同道德后果 |
| 义士 × 编队短板 | 你缺远程→解救的义士是弓手→补短板。义士的类型每局随机（或每关固定），制造"这局救了谁"的 roguelike 变量 |
| 内应 × 时迁 | 时迁本来只是个"绕墙切核心"的角色。加内应系统后→时迁多了一层玩法：找内应、拿情报。而且内应标记只有侦查期可见→玩家需要在 8s 内记住内应位置，或者让时迁逐一验证 |

### 4.2 敌方士气系统

**机制定义**

敌方单位有士气值（0-100），士气归零时该单位**溃逃**（脱离战斗，向地图边缘逃窜，不计入击杀数，不触发遗计/嗜血）。

| 事件 | 士气影响 | 范围 |
|------|---------|------|
| 看到 Boss 守将被击杀 | -50 | 全场 |
| 看到身边庄丁被击杀 | -15 | 周围 3 格 |
| 看到李逵击杀（恐惧光环） | -25（额外） | 李逵周围 5 格 |
| 看到援军抵达 | +20 | 全场 |
| 看到烽火升起（己方求援） | +10 | 全场 |
| HP < 30% | -20 | 自身 |
| 听到吴用"劝降"技能（新增） | -30 | 吴用周围 4 格 |

**涌现点**

| 涌现交互 | 说明 |
|---------|------|
| 士气 × 击杀优先级 | "先杀小兵削弱士气，还是先斩首 Boss 让其溃散？" Boss 被击杀 = 全场 -50→可能导致半数庄丁溃逃→战斗提前结束。但 Boss 本身是最难打的。风险回报涌现 |
| 士气 × 李逵 | 李逵的"杀红眼"被动＝击杀后攻速叠加。如果李逵造成溃逃而非击杀→不触发杀红眼。李逵玩家需要考虑：我是追着溃逃的庄丁补刀叠层，还是让他们跑？ |
| 士气 × 吴用新技能 | 给吴用加一个"劝降"主动技：对低士气庄丁发动（士气 < 30），概率使其倒戈→变成友方单位（不占部署槽）。吴用从"召唤填线"扩展为"策反敌军" |
| 士气 × 警报等级 | 警报 3 时敌方有援军→士气恢复→不容易溃逃。但如果玩家在警报 0 时制造大量击杀→敌方还没等到援军就溃散了。 |
| 士气 × 劫掠 | 庄丁溃逃时可能丢弃装备（概率掉落），玩家选择：追杀抢装备 vs 直接推核心 |

### 4.3 战场事件：梁山号令

**机制定义**

每局随机触发 1-2 次"梁山号令"事件（名正言顺的援军/天意）。

| 号令 | 触发条件 | 效果 | 持续 |
|------|---------|------|------|
| ⚡ **天罡星显** | 己方单位 HP < 20%（任一） | 该单位 HP 回满 + 移速翻倍 | 8s |
| 🌪️ **黑旋风起** | 李逵击杀数 ≥ 5 | 李逵进入狂暴：攻速 +100%、不受控制（自动索敌）、持续自伤 | 10s |
| 🏹 **神箭天降** | 花荣未命中 ≥ 3（被闪避） | 下次百步穿杨必命中 + 伤害 ×2 | 一次性 |
| 🔥 **替天行道** | 拨济比例选最高档 | 全场敌方士气 -30 + 己方全体攻速 +10% | 15s |

**涌现点**

号令不是"大招"，而是**消耗你已有资源（HP低位、李逵血量、拨济比例）换取的战术窗口**。你需要主动创造触发条件，而不是被动等号令。

## 7. 优先级与实施路线

### 7.1 机制评级（影响力 × 实现成本）

| 机制 | 影响力 | 实现成本 | 评级 | 建议 |
|------|--------|---------|------|------|
| 警报等级 | ★★★★★ | ★★★ | S | **第一优先**——这是所有动态防御机制的基础层 |
| 哨兵 | ★★★★★ | ★★ | S | **第一优先**——低成本高回报，创造大量涌现交互 |
| 巡逻队 | ★★★★ | ★★ | A | 和哨兵一起上线，两者搭配效果更好 |
| 战争迷雾 | ★★★★★ | ★★★★ | A | 影响巨大但需要较完整的视野系统 |
| 可交互环境物 | ★★★★ | ★★★ | A | 火药桶/烽火台 ROI 极高 |
| 庄丁兵种分化 | ★★★ | ★★★ | B | 锦上添花，可在警报系统之后做 |
| 天气系统 | ★★★★ | ★★★★ | B | 表现层工作量大（粒子特效），但设计空间极大 |
| 指挥官 AI | ★★★★ | ★★★★★ | B | 需要效用评估 AI，实现复杂度最高 |
| 士气系统 | ★★★ | ★★★ | B | 依赖庄丁兵种分化后才能发挥 |
| 昼夜系统 | ★★★ | ★★ | B | 可以简化为"黑夜模式"toggle，不与天气叠加 |
| 平民/内应 | ★★ | ★★ | C | 叙事价值高但玩法影响有限 |
| 英雄新技能 | ★★★★ | ★★ | A | 配合新机制调整现有技能，必须做 |

### 7.2 三阶段落地建议

**阶段一：MVP 增强（做 3-4 个机制，让基础好玩）**

1. **警报等级系统**（§2.1）——所有动态防御的地基
2. **哨兵**（§1.2）——最小信息战单元
3. **巡逻队**（§1.3）——让战场"活"起来
4. **英雄技能微调**（§6）——以上机制所需的技能修改

这一阶段不做迷雾、不做天气、不做指挥官 AI。只用哨兵+巡逻队+警报等级，就让现有设计从"静态拼图"变成"动态博弈"。

**阶段二：涌现深化（加 3-4 个机制，让每局不同）**

5. **战争迷雾**（§1.1）——让信息变成稀缺资源
6. **可交互环境物**（§3.3）——火药桶、烽火台、酒坛
7. **天气系统**（§3.1）——最低成本的环境变量
8. **庄丁兵种分化**（§2.3）——让敌人也有性格

**阶段三：长尾留存（加 2-3 个机制，让深度玩家有东西挖）**

9. **指挥官 AI**（§2.2）——每个关卡的指挥官性格不同
10. **士气系统**（§4.2）——让"不战而屈人之兵"成为可能的胜利路径
11. **平民/内应/昼夜**（§3.2 + §4.1）——叙事与氛围

## 8. 反漂移校验：新机制是否破坏设计支柱？

| 支柱 | 新机制影响 | 是否漂移 |
|------|-----------|---------|
| P1 以少胜多 | 警报系统让"少"更有价值（低调 = 警报低）；12 上限未变 | ✅ 无漂移 |
| P2 组合策略 | 天气/环境/哨兵创造更多 combo 变量，组合空间指数增长 | ✅ 增强 |
| P3 地形博弈 | 迷雾+哨兵巡逻队+可交互物让"读图"从静态变动态博弈 | ✅ 增强 |
| P4 劫富济贫 | 平民系统强化救济叙事；警报等级影响劫掠决策 | ✅ 增强 |

没有漂移。新机制全部增强现有支柱，而非削弱或偏离。

## 9. 总结：涌现引擎

所有这些机制的共同目标不是"加更多玩法"，而是构建一个**涌现引擎**：

```
┌─────────────────────────────────────────────┐
│              涌现引擎核心循环                │
│                                             │
│  玩家选择 → 改变战场状态 → 敌方反应 →        │
│       ↑                          ↓          │
│       └── 新信息/新约束 ←── 状态改变 ←──┘    │
│                                             │
│  每循环一次，战场状态都不可预测地演化         │
│  没有两局相同                                │
└─────────────────────────────────────────────┘
```

具体的"涌现引擎组件"：

| 组件 | 提供什么变量 | 变量怎么变 |
|------|------------|-----------|
| 战争迷雾 | 信息完整性 | 玩家主动侦察→信息增加；被哨兵发现→敌方信息增加 |
| 哨兵+巡逻队 | 被发现风险 | 路线随机偏移+玩家干扰→行为不可预测 |
| 警报等级 | 敌方强度 | 累积升高、不可逆、影响所有敌方行为 |
| 天气 | 全局规则 | 开局随机、可被技能改变、覆盖所有单位 |
| 环境物 | 战术选项 | 一次性消耗、用法互斥、部分可被敌方抢先 |
| 指挥官 AI | 敌方适应性 | 每 15-20s 根据已知信息做一次决策 |
| 士气 | 敌方稳定性 | 累积下降、可被击杀/Boss 阵亡/技能操控 |

当所有这些变量同时活跃时，一局 120s 的战斗中，系统的可能状态数远超设计师能枚举的范围。这就是涌现。
