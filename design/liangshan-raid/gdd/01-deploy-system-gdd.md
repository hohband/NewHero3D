# GDD-01 部署系统（Deploy System）

> **所属模式**：劫寨战斗模式（Raid the Stronghold）· Phase 2 系统设计
> **上游文档**：`../combat-mode-design.md`（DS-001，§0.1 已审批锁定 L1/L2/L3）
> **设计支柱对齐**：P1 以少胜多（兵符硬预算）、P3 地形博弈（侦查期读图）、R1（兵符硬上限）、R2（在场 12 性能护栏）、R6（认知过载：1.5s 间隔 + 侦查期）
> **架构红线**：本系统为真·实时部署，**不采用 SRPG 的 `BattleManager.submit_command()` 回合模型**；部署请求由 `RealTimeBattleManager.requestDeploy()` 处理。

---

## ① 概述（Overview）

部署系统负责把玩家在「梁山泊大营」生成的武将送入战场，并管控三类约束：**三重资源预算**（兵符 / 粮草 / 声望）、**部署节奏与冷却**、**编队槽与带兵护栏**。它是进攻方"主动出手"的唯一入口，直接决定"何时、何地、放谁"。

- 单位以**生成（spawn）**方式入场，部署点 = 生成锚，非移动入场。
- 命名 ✅ 已锁定：兵符（战术层每局预算）/ 粮草（实时层技能消耗）/ 江湖声望（Meta 层解锁，不进战斗数值）。
- 护栏 ✅ 已锁定：8 编队槽 / 同场同名武将限 1 / 实时在场上限 12。
- **v2 同步**：撤兵机制、节奏档位/直控/战术指令、粮草通用约束、召唤独立计数、暂停语义统一 —— 见 `../combat-mode-design.md` §0.2 / §2.6。

## ② 机制（Mechanics）

1. **出兵点（Spawn Point）**
   - 位置：战场边缘、敌方外墙之外的"安全区"，不可被攻击/摧毁。
   - 数量：默认 1 个；L3+（大名府起）开放 **2–3 个部署点**，增加多向开花战术维度。
   - **v2 劫营点（高风险部署）**：部分地图提供 1 个敌腹地"劫营点"（如粮仓旁），部署于此的单位立即获攻击 +30%、移速 +20%，但初始吸引所有塔仇恨（combat-mode §2.1）。
2. **三重资源预算**
   - **兵符**：每关固定预算（建议 30）。武将按稀有度消耗 —— 普通 3 / 精英 6 / 传奇 10。预算用尽则不可再部署新单位。
   - **粮草**：每关固定预算（**v2 建议 150**，原 100 过紧）。**所有主动技能均耗粮草 5–30**（v2 改口径，原仅 3 将），粮草耗尽 → 主动技能禁用，被动/光环/怒气大招不受影响（见 GDD-02 §3.3）。
   - **声望**：Meta 层，不进本局数值（见 GDD-04）；Meta 层另用于购买**兵书**（永久微调，combat-mode §0.2.2）。
3. **部署节奏**
   - 部署间隔（Deploy Interval）：默认 **1.5s**，防瞬间铺满、降认知过载（R6）。
   - 再部署冷却（Re-deploy CD）：被撤兵或"重伤退场"（叙事包装，见 combat-mode T2）的武将需冷却（建议 **20s**）才能再次上场。
4. **编队槽（Squad Slots）**
   - 开战前编排上场顺序，最多 **8 槽**；战斗中按序或自由部署。
   - 不强制阵型，但提供预设槽以降开局决策压力。
5. **侦查期（Scout Phase）**
   - 开战前 **8s** 可自由旋转/缩放摄像机、标记防御，**不可部署**；满足 SDT 自主并降信息压力（R6）。
   - **v2 标记持久（M6）**：侦查期标记**永久保存至战斗结束**，层 3 默认折叠可一键展开（防凭记忆打陷阱）。
6. **战术时间控制（PC 补偿）**
   - 战术暂停（`Space`）/ 0.5x 慢放（`Shift`）：暂停/慢放期间不累计部署冷却与战斗计时（具体语义由 RealTimeBattleManager 统一，见 §⑦）。
   - **v2 暂停上限（P1-11）**：暂停次数上限 **3 次（每次 ≤15s）** 或暂停总时长 ≤60s；0.5x 慢放总时长 ≤30s（防实时紧张感被无限思考瓦解，combat-mode U1）。
7. **v2 撤兵机制（Retreat，落 P0-4）**
   - **触发**：选中在场单位 → 点"撤兵"按钮或快捷键 `X`。
   - **效果**：单位立即离场；兵符**不返还**（沉没成本）；计入 `alive`（否则"撤兵换星"逻辑不成立，见 GDD-04 §④）。
   - **再部署冷却**：撤兵单位进入 20s 再部署冷却（与重伤退场同处理）。
   - **边缘**：撤兵不可撤销；若单位正处于嘲讽/光环中，效果立即失效。
8. **v2 节奏档位 / 直控 / 战术指令（combat-mode §2.6）**
   - **节奏档位（开局选）**：导演档（轻操作+全局指令）/ 指挥官档（直控 1 先锋+少量指令）/ 猛将档（直控+全指令）。
   - **直控层（指挥官/猛将档）**：玩家微操 1 名"先锋"武将（WASD/点移+技能），其余自动 AI；先锋重伤退场则控制权转交下一名存活将。
   - **战术指令（导演/指挥官档）**：集火(F)/后撤(G)/冲锋(H)/坚守 —— 短冷却 8–15s、耗粮草 5–10，覆盖单位 AI 而非取代。

**硬性护栏（✅ 已锁定）**
- 单场带兵槽位：8。
- 同场同名武将上限：1（防单卡 stack 主导策略）。
- 实时同时在场单位上限：12（性能护栏；超出则最老单位不可再部署，直到场上数量下降）。
- **v2 召唤物独立计数（M4）**：吴用援军用独立 `summon_cap = 4`，与 `live_unit_cap = 12` **解耦**，避免惩罚吴用玩家；粮草一次性消耗不持续。

**v2 编队示例（P0-6，证明预算与槽位张力有趣）**
> 假设稀有度：传奇 10 / 精英 6 / 普通 3，兵符预算 30。
> - 3 传奇 + 0 精英 + 0 普通 = 30（3 槽，5 槽空）
> - 2 传奇 + 1 精英 + 1 普通 = 29（4 槽）
> - 1 传奇 + 3 精英 + 0 普通 = 28（4 槽）
> - 0 传奇 + 5 精英 + 0 普通 = 30（5 槽）
> - 0 传奇 + 4 精英 + 2 普通 = 30（6 槽）
> 实际可行编队约 20–40 种；**v2 以武魂附身（combat-mode §3.4）在不扩 roster 前提下把 combo 空间扩到 ~3480，化解"8 选 8 仅 1 种"问题**。

## ④ 公式（Formulas）

设：`B_bf` = 兵符预算，`c_i` = 第 i 次部署武将的兵符消耗，`U_bf = Σ c_i`（已部署累计）。

- **兵符剩余**：`R_bf = B_bf − U_bf`
- **可部署（兵符）**：`R_bf ≥ c_target`
- **再部署许可**：`(now − t_last_end[hero]) ≥ redeploy_cd_s` **且** `R_bf ≥ c_target`
  - 注：兵符在每次部署事件消耗（含再部署），故预算同时约束总部署次数与护栏。
- **实时在场许可**：`N_live < live_unit_cap`（`N_live` = 当前激活单位数）
- **同名在场许可**：`count(active units with hero_id == h) < same_name_cap`（即 0，cap=1）
- **v2 综合部署许可判定顺序（M2，强制）**：`再部署冷却检查 → 同名在场检查 → 兵符预算检查 → 在场上限检查 → 非侦查期 → 非暂停禁止态`。顺序固定，防"死帧绕过 20s 冷却"。
- **v2 撤兵许可（§②.7）**：选中在场单位 + 非暂停禁止态 ⇒ 允许撤兵；撤兵不返还兵符、计入 `alive`、进入 20s 再部署冷却。
- **侦查期约束**：`is_scout_phase == true ⇒ 拒绝所有 requestDeploy`

## ⑤ 边缘情况（Edge Cases）

1. **兵符边界**：`R_bf < c_target` 时该武将不可部署（编队栏置灰），已部署单位继续战斗，不报错、不扣减。
2. **在场上限边界**：`N_live == 12` 时请求部署 → 拒绝并提示"在场已达上限"；最老单位阵亡后自动恢复许可。
3. **再部署冷却边界**：同名武将 20s 内重复部署 → 拒绝，编队栏显示冷却倒数，不消耗兵符。
4. **侦查期误触**：侦查期 8s 内点击/拖拽部署 → 忽略输入，保持"可标记不可部署"状态。
5. **粮草耗尽**：粮草=0 时主动技能禁用（被动/光环/怒气大招仍生效），HUD 兵符/粮草分区提示缺口。
6. **战术暂停/慢放（M3 统一语义）**：**暂停期间禁止部署与技能释放，仅允许摄像机操作与查看编队栏/状态**（统一 GDD-01 §⑤.6 与 GDD-04 §⑤.7）；不累计冷却、不计入战斗用时（`elapsed`）。暂停受 §②.6 次数/时长上限约束。
7. **多部署点切换**：L3+ 多点时，落点必须属于已激活部署点半径内，否则拒绝并高亮有效点。
8. **v2 撤兵边界（§②.7）**：撤兵不可撤销；撤兵时若单位处于嘲讽/光环中，效果立即失效；撤兵单位计入 GDD-04 `alive`（Star3 计算）。
9. **v2 召唤物上限（M4）**：吴用援军占用独立 `summon_cap=4`，不挤占 `live_unit_cap=12`；援军持续期间粮草一次性消耗不持续。

## ⑥ UI 接口（UI Interface — 对齐 §7.5 / §7.6 HUD）

- **编队栏（底部）**：8 槽显示待部署武将头像 + 兵符消耗；不可部署（冷却/兵符不足/同名在场）置灰并显示原因。
- **资源条（层 1 常驻）**：兵符剩余 / 粮草剩余 / 在场数 `N/12` / 核心血量 / 计时。
- **落点预览圈**：拖拽武将到战场时显示射程 / 索敌预览圈（§7.5）。
- **快捷键 1–8**：快速选中待部署武将（§7.5）。
- **战术暂停 `Space` / 0.5x 慢放 `Shift`**（§7.5，受 §②.6 上限约束）。
- **撤兵键 `X`**（§②.7）；战术指令集火(F)/后撤(G)/冲锋(H)/坚守（§②.8）。
- **节奏档位选择**（开局）：导演/指挥官/猛将（§②.8）。
- **侦查期标记（层 3 可折叠，v2 永久保存）**：旋转/缩放读图，标记墙/塔/陷阱，展开详细防御情报；**标记永久保留至战斗结束，默认折叠可一键展开（M6）**。

## ⑦ 依赖（Dependencies）

- **跨系统**：
  - GDD-02 武将技能系统：兵符消耗定义、英雄属性、粮草预算共用（v2 全主动技耗粮草）、怒气/武魂/关系模块。
  - GDD-03 基地与防御系统：部署点位置、墙阻挡影响单位寻路、在场上限约束、劫营点（v2）、动态防御响应。
  - GDD-04 经济与评分系统：第三星"剩余兵力"判定取存活/已部署兵符比（v2 按英雄 ID 去重）、资源命名一致。
- **Phase 3 接口**：
  - `RealTimeBattleManager.requestDeploy(hero_id, spawn_point_id, target)` → 校验预算/冷却/在场/同名/侦查期 → 生成单位；**非 SRPG submit_command 回合模型**。
  - v2 新增：`requestRetreat(hero_id)`（撤兵）、`setRhythmTier(tier)`（节奏档位）、`directControl(hero_id)`（直控先锋）、`issueOrder(order_type)`（战术指令）。
  - 固定步长逻辑层（建议 20Hz）+ 表现层插值；流场寻路（R2，分块重算）；`RollSource` 注入随机（模板偏移在 GDD-03；闪避/暴击/陷阱在 GDD-02）。
  - 暂停/慢放语义由管理器统一广播给所有系统（含冷却与计时），且受 §②.6 上限约束。

## ⑧ 验收标准（Acceptance Criteria）

- [ ] 8 槽编队可编排，1–8 快捷键正确选中待部署武将。
- [ ] 兵符预算耗尽时对应武将正确禁用，不误扣、不崩溃。
- [ ] 粮草预算耗尽时主动技能禁用，被动/光环仍生效。
- [ ] 同名武将在场限 1、实时在场上限 12 强制生效。
- [ ] 部署间隔 1.5s、再部署冷却 20s 计时准确（暂停不累计）。
- [ ] 侦查期不可部署但可标记/旋转/缩放。
- [ ] 落点预览圈与实际射程/索敌一致。
- [ ] L3+ 多部署点正确激活并可切换落点。
- [ ] **v2 撤兵机制**：`X` 触发、兵符不返还、计入 alive、进入 20s 冷却、不可撤销。
- [ ] **v2 部署许可顺序固定**：再部署冷却 → 同名 → 兵符 → 在场（M2）。
- [ ] **v2 暂停语义统一**：暂停禁止部署/技能，仅摄像机+查看（M3）。
- [ ] **v2 召唤独立计数**：吴用援军用 `summon_cap=4` 不挤占在场 12（M4）。
- [ ] **v2 节奏档位/直控/战术指令**三档可选且正确生效。
- [ ] CSV 字段完整，数值热更后实时生效，无硬编码。
