# GDD-02 武将技能系统（Hero Skill System）

> **所属模式**：劫寨战斗模式（Raid the Stronghold）· Phase 2 系统设计
> **上游文档**：`../combat-mode-design.md`（DS-001，§3 八将体系、§0.1 已审批锁定 L1 资源命名）
> **设计支柱对齐**：P2 技能组合即策略（Combo-Driven）、P1 以少胜多（技能放大而非数量）、R3（技能数值膨胀）、R4（与 CoC 差异化 combo）
> **架构红线**：技能释放为实时手动触发，经 `RealTimeBattleManager.castSkill()`；**不采用 SRPG 的 `submit_command()` 回合模型**。增益/减益以"状态组件（Status Component）"实现，支持叠加与驱散。

---

## ① 概述（Overview）

武将技能系统定义 8 名梁山武将的**属性模板**与**技能**（主动 / 被动 / 光环），是 P2「Combo-Driven」支柱的落地核心。技能由玩家实时手动触发，受**粮草**消耗与**冷却**约束；被动/光环不受粮草影响。8 将技能设计为可串联的"化学反应"（嘲讽→AOE、召唤→破甲、潜行→狙杀），而非各自为战。

- 资源命名 ✅ 已锁定：粮草（实时消耗）、兵符（编制限制，见 GDD-01）、声望（Meta，见 GDD-04）。
- 防数值膨胀（R3）：技能 impact 与冷却成反比曲线，且受粮草预算约束；L4 法术塔驱散反制（v2 改为周期脉冲）。
- **v2 同步**：全主动技耗粮草、怒气系统、武魂附身、关系羁绊、技能变体 —— 见 `../combat-mode-design.md` §0.2 / §3.3–§3.7。

## ② 机制（Mechanics）

1. **属性模板（每将）**：HP / DPS / 移速 / 攻击范围 / 破墙系数 / 标签（肉盾/突进/远程/潜行/狂暴/AOE）。
2. **技能三类**：
   - **主动**：手动触发，消耗粮草 + 冷却（如吴用七星聚义耗粮草 30 / 冷却 22s）。
   - **被动**：常驻生效，不受粮草影响（如燕青相扑 +30% 闪避）。
   - **光环**：范围增益，持续生效（如吴用智多星友军攻速 +15%）。
3. **冷却错峰**：技能冷却分布 10–22s 不等，鼓励"链"而非"同时全开"。
4. **Combo 接口**：每个武将至少一个"被他人增益放大"或"放大他人"的接口：
   - 鲁智深嘲讽聚怪 → 公孙胜五雷天罡 AOE 清场；
   - 吴用召唤填线 → 林冲破甲 → 公孙胜补 AOE；
   - 时迁潜行绕墙 → 花荣狙掉守核心塔。
5. **单位 AI**：部署后自动寻路/索敌/攻击（战斗主循环），技能为玩家叠加的操作层；指挥官/猛将档可直控 1 先锋（GDD-01 §②.8）。
6. **防膨胀（R3）**：`impact_scale = k × sqrt(cooldown_s)`（平方根曲线，v2 采纳 review N4，比原反比更平缓），且总释放次数受粮草预算（150）上限约束。
7. **v2 怒气系统（§3.6）**：普攻/受击积攒怒气（每将独立条）；满怒气可放"绝技"（无视冷却、耗粮草）；怒气可"借"给队友触发其技能提前/增伤，借出者 5s 虚弱（下 1 普攻 +100%）。
8. **v2 武魂附身（§3.4）**：每局开战前从 ~30 武魂池选 2 附身主将，仅 1 被动 + 1 小技，不增 roster 记忆负担，组合空间 8×C(30,2)≈3480。
9. **v2 关系值 + 羁绊（§3.5）**：8 将间定义关系，战斗中累积关系值；达阈值触发叙事性羁绊技能（非预设 combo，是关系触发的戏剧时刻）。
10. **v2 技能变体（战场专长，§3.7）**：每局每名上场武将从 3–4 个 `variant_pool` 随机带 1，制造局内随机惊喜（与武魂的玩家选择互补）。

## ④ 公式（Formulas）

- **技能可用（主动）**：`(now − t_last_cast[hero]) ≥ cooldown_s` **且** `R_lc ≥ liangcao_cost`
  - `R_lc = B_lc − Σ 已释放主动技粮草消耗`（粮草预算见 GDD-01）
- **嘲讽（鲁智深·罗汉金身）**：周围敌方强制索敌自身免疫，持续 `taunt_duration = 4s`；友军减伤 `25%`。
- **攻速增益（吴用·智多星）**：友军 `attack_speed × (1 + 0.15)`。
- **破甲（林冲·风雪山神庙）**：目标 `defense × (1 − 0.40)`，持续 `3s`。
- **破墙系数（李逵·板斧旋风）**：对墙 `breach_coeff × 2`。
- **Combo 链实现**：以"状态组件"叠加表达（嘲讽态 → AOE 增伤态），非硬编码释放顺序；增益可被 L4 法术塔驱散清空。
- **防膨胀约束（v2）**：`impact_scale = k × sqrt(cooldown_s)`（平方根曲线，N4），且 `Σ liangcao_cost ≤ B_lc(150)` 限制总释放次数。
- **v2 怒气公式**：`rage += d_rage_per_hit`（普攻/受击累积）；`rage_full ⇒ castUltimate()`（无视 `cooldown_s`，仍耗 `liangcao_cost`）；`borrowRage(from, to)`：from 清空怒气 → to 技能提前释放/增伤，from 进入 5s 虚弱（`next_attack_dmg × 2`）。

## ⑤ 边缘情况（Edge Cases）

1. **粮草不足**：主动技禁用，HUD 显示粮草缺口，不扣减、不触发。
2. **冷却中释放**：忽略输入，技能栏显示冷却倒数。
3. **嘲讽目标失效**：嘲讽目标已阵亡/离场 → 重新选最近有效敌方或效果失效，不报错。
4. **增益光环内无友军**：光环空转，不报错、不耗能。
5. **召唤物越在场上限**：吴用援军用独立 `summon_cap = 4`（GDD-01 §②，M4），不占 `live_unit_cap = 12`；超则拒绝生成。
6. **潜行被驱散（L4 法术塔）**：时迁潜行态被驱散脉冲命中 → 立即显形并进入塔/守军索敌（v2 驱散为周期脉冲，可被 timing/诱饵骗出，combat-mode §4.6）。
7. **被动/光环不受粮草影响**：粮草=0 时被动/光环/怒气大招仍正常生效（与主动区分）。
8. **v2 武魂/关系边界**：武魂附身仅 1 被动+1 小技，不替代主技能；关系羁绊达阈值才触发，非每局必现；怒气借出后虚弱期不可再借。
9. **v2 RollSource 明示（H3）**：闪避/暴击/陷阱触发等概率判定经 `RollSource` 注入（生产可设种子），不直用 Math.random。

## ⑥ UI 接口（UI Interface — 对齐 §7.5 / §7.6 HUD）

- **技能栏 Q / E / R 或全局技能轮盘**：选中单位显示技能图标 + 冷却环 + 粮草消耗（层 2）。
- **落点/目标预览**：指定区域技能（公孙胜）显示范围圈；方向技能（林冲）显示箭头；目标技能（花荣）显示锁定线。
- **资源条（层 1 常驻）**：粮草剩余实时显示；主动技粮草不足置灰。
- **状态图标（层 2）**：增益/减伤/嘲讽/潜行/驱散显示在受影响单位头顶。
- **v2 怒气条（层 2）**：每将独立怒气条；满怒气高亮提示可放绝技；借出动画。
- **v2 武魂/关系标识**：附身武魂图标、关系值进度、羁绊技能就绪提示。

## ⑦ 依赖（Dependencies）

- **跨系统**：
  - GDD-01 部署系统：粮草预算共用、英雄属性、在场上限约束召唤物。
  - GDD-03 基地与防御系统：塔索敌受嘲讽改变（§4.4）、破墙系数作用于墙、L4 法术塔驱散克制增益/召唤。
  - GDD-04 经济与评分系统：粮草命名一致、声望 Meta 解锁武将。
- **Phase 3 接口**：
  - `RealTimeBattleManager.castSkill(hero_id, skill_id, target)` → 校验冷却/粮草 → 施加状态组件；**非 SRPG submit_command**。
  - v2 新增：`castUltimate(hero_id)`（怒气绝技）、`borrowRage(from,to)`（怒气交换）、`attachSoul(hero_id, soul_a, soul_b)`（武魂附身）、`updateRelationship()`（关系值累积）。
  - 状态组件系统：支持施加/叠加/驱散；驱散事件由防御系统（法术塔脉冲）广播。
  - 固定步长 20Hz 逻辑层驱动冷却与持续效果计时；**所有概率判定（闪避/暴击/陷阱/武魂随机/变体抽取）经 `RollSource` 注入（H3）**。
