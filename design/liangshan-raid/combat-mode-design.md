# 劫寨战斗模式设计文档（Combat Mode: "Raid the Stronghold"）

> **项目**：水浒题材独立游戏（PC / 3D 网页，表现层 Three.js 2.5D）
> **战斗参考**：Clash of Clans（部落冲突）实时部署（Real-Time Deployment）
> **本文范畴**：仅设计"劫寨"这一战斗模式本身 —— 非完整游戏概念、非美术圣经（Art Bible）、非代码。
> **阶段衔接**：本文是 Phase 1 概念产出，直接喂给 Phase 2（系统设计 / GDD 八节）与 Phase 3（技术搭建）。
> **设计理论锚点**：MDA 框架、动词优先法（Verb-First）、自我决定论（SDT）、心流（Flow）、Bartle 玩家类型。
> **设计红线（贯穿全文）**：杜绝主导策略、经济失衡、认知过载、支柱漂移；发现即标注 ⚠️。
> **版本**：v2（2026-07-21 整合 8 份评审/改进文档后的取舍修订版）。

## 0. 模式一句话定义

玩家以**梁山进攻方**身份，从**梁山泊大营**逐个部署武将，武将自动寻路/索敌/攻击，技能由玩家实时手动触发；目标是突破敌方山寨（祝家庄等）的**墙—塔—陷阱—守军**防御体系，**击败守寨头领（Boss 守将）并摧毁核心建筑**并**劫掠资源**，收益通过"劫富济贫"反哺江湖声望与剧情，形成题材闭环。

> **v2 关键修订**：核心胜利判定从"纯建筑核心"升级为"守寨头领（Boss 守将） guarding core"——让"对面有脑子"与"反派有脸"成为设计前提（见 §4.9、§6）。

## 1. 核心循环与设计支柱

### 1.1 核心循环（Core Loop）

```
侦察(Scout) → 部署(Deploy) → 触发技能(Cast) / 直控先锋(Direct) / 发令(Order) / 攒怒气(Rage)
     → 自动结算(Resolve: 寻路/拆墙/索敌/攻击) ↔ 动态防御响应(Alert/Sentry/Commander)
     → 劫掠(Loot, 实时动词) → 评分(Score: 三星) → 拨济(Relief, 叙事分支) → 声望/剧情解锁(Meta)
```

单局节奏（建议）：先手侦查期 8s → 实时部署与交战 60–180s（含 Boss 战高潮）→ 结算 5s。单局时长 **1.5–4 分钟**（L1 教学约 1.5 分钟，L5 终局约 4 分钟，修正原"2–4 分钟"下限不一致，见 U2）。

### 1.2 MDA 框架

**Mechanics（机制层 — 系统提供的基础规则）**
- 部署系统：出兵点、兵符预算、部署间隔冷却、编队槽、再部署冷却、撤兵。
- 单位 AI + 直控层：自动寻路（绕墙/破墙）/ 索敌 / 攻击；玩家可直控 1 名先锋；战术指令覆盖 AI。
- 防御结构：外墙/内墙/箭塔/瞭望塔/法术塔（驱散脉冲）/陷阱/庄丁（6 兵种）/核心 + Boss 守将。
- 动态防御：警报等级（0–4，只升不降）/ 哨兵 / 巡逻队 / 指挥官 AI（有限 CP 取舍）。
- 技能系统：主动（手动触发，耗粮草+怒气）/ 被动（常驻）/ 光环（范围增益）/ 武魂附身 / 关系羁绊 / 怒气交换。
- 资源经济：兵符（编制限制）/ 粮草（实时消耗，所有主动技通用约束）/ 江湖声望（Meta 解锁 + 兵书）。
- 环境：天气（晴/雨/雾/风/雪）/ 昼夜 / 可互动物（火药桶/烽火台/酒坛/吊桥）。
- 单位属性维度：HP、DPS、移速、攻击范围、破墙系数、标签（肉盾/突进/远程/潜行/狂暴/AOE）。

**Dynamics（动态层 — 机制交互涌现出的玩法）**
- **实时决策涌现**：何时/何地/放谁/放哪个技能/直控走位/发哪道令/攒还是交怒气，每秒产生分支。
- **以少胜多**：带兵上限压低，胜利依赖路线/时机/combo/欺骗，而非数量碾压。
- **技能化学反应**：状态组件自由组合 + 武魂附身 + 关系羁绊 + 敌方警报/天气/地形构成的"变量×变量"组合爆炸。
- **地形与信息博弈**：墙/塔/陷阱/迷雾/哨兵构成"可读且会变"的谜题；主动欺骗（诱饵/佯退）让"智取"可操作。
- **风险回报**：撤兵保星 vs 全押；远程点塔（安全慢）vs 近战破墙（快但脆）；劫营高风险部署；破釜沉舟。
- **对面有脑子**：敌方按"你在做什么"反应（警报升级、指挥官调兵、庄丁增援被破墙方向），是博弈而非解题。

**Aesthetics（美学 / 体验目标）**
- **挑战感（Challenge）**：以少胜多的智取快感。
- **发现感（Discovery）**：拆穿敌方布局、迷雾中摸清伏兵、找到防御薄弱点。
- **表达感（Expression）**：武魂 build / 兵书 / 关系羁绊 / 节奏档位体现个人风格。
- **叙事沉浸（Narrative）**：劫富济贫让掠夺有正向意义；Boss 守将、武将高光时刻、拨济分支让"打家劫舍"有水浒魂。

### 1.3 设计支柱（Design Pillars — 4 条，所有后续决策以此为准绳）

| # | 支柱 | 一句话 | 反漂移校验 |
|---|------|--------|-----------|
| P1 | **以少胜多（智取 > 堆量）** | 带兵上限压低，靠路线/时机/combo/欺骗取胜 | 若某关靠"堆满 12 单位平推"可过 → 支柱漂移 ⚠️ |
| P2 | **技能组合即策略（Combo-Driven）** | 武将技能 + 武魂 + 状态组件设计为可串联的化学反应 | 若单卡无配合也能通全关 → 支柱漂移 ⚠️ |
| P3 | **地形博弈（读图 > 莽）** | 墙/塔/陷阱/迷雾构成可读且动态的谜题 | 若绕开读图也能盲打通关 → 支柱漂移 ⚠️ |
| P4 | **劫富济贫的题材闭环（Loot-and-Give）** | 劫掠（实时动词）反哺声望/剧情，掠夺有正向意义 | 若救济纯惩罚无收益 → 支柱漂移 ⚠️ |

> **支柱漂移红线**：后续任何机制（含数值、关卡、技能）若导致以上任意支柱失效，必须回退或重新平衡。v2 新增的动态防御/武魂/怒气/劫掠动词均经 §8 反漂移校验，结论为**增强而非削弱**。

### 1.4 动词优先法（Verb-First — 核心玩法动词）

| 动词 | 含义 | 对应系统 |
|------|------|----------|
| **部署** Deploy | 选点放下武将 | 部署系统 |

## 2. 进攻方部署机制

### 2.1 出兵点（Spawn Point）
- **名称**：**梁山泊大营（Liangshan Camp）**。
- **位置**：战场边缘、敌方外墙之外的"安全区"，本身不可被攻击、不可被摧毁。
- **数量**：默认单点；高级关卡（§6 L3+）可开放 **2–3 个部署点**以增加战术维度（多向开花）。
- **本质**：单位是"生成"而非"移动入场"，部署点 = 生成锚。
- **v2 新增·劫营点（高风险部署）**：部分地图提供 1 个位于敌腹地的"劫营点"（如粮仓旁），部署于此的单位立即获攻击 +30%、移速 +20%，但初始吸引所有塔仇恨（analysis 杠杆五·劫营点，低成本戏剧性，采纳）。

### 2.2 兵力 / 资源系统（三重资源 · ✅ 命名已审批锁定）
采用**三重资源**，刻意分离，避免单一资源导致策略单薄（防主导策略）：

| 资源 | 命名 | 作用层 | 用途 | 防失衡说明 |
|------|------|--------|------|-----------|
| **兵符** Troop Tally | 兵符 | 战术层（每局） | 部署武将消耗的"编制"，决定每场可带武将总量与品质上限 | 硬预算上限，杜绝"囤兵一波流" ⚠️ |
| **粮草** Provisions | 粮草 | 实时层（每局） | **所有主动技能**的通用消耗（v2 改口径，见 §3.3）+ 召唤援军 | **通用战术约束**，不再只约束 3 将（修正 N2） |
| **江湖声望** Jianghu Renown | 声望 | Meta 层（跨局） | 通过劫掠救济积累，解锁更高级武将 / 技能 / 关卡 / 兵书 | 进度解锁，不进战斗数值 |

> **v2 粮草口径修订（采纳 N2）**：原设计仅吴用/公孙胜/花荣耗粮草，粮草对不带这 3 将的玩家完全无效，违背"通用战术约束"定位。改为**所有主动技均耗粮草（5–30 区间）**，使粮草成为每局都参与的取舍资源；预算由 100 上调至 **150**（GDD-01 §③）以容纳更密释放。被动/光环仍不受粮草影响。

### 2.3 部署节奏与冷却
- **部署间隔（Deploy Interval）**：默认 **1.5s**（可调），防止瞬间铺满全场、降低认知过载。
- **再部署冷却（Re-deploy CD）**：被撤兵或"重伤退场"（叙事包装，见 T2）的武将需冷却（建议 **20s**）才能再上场，鼓励"先想后放"。
- **兵符预算**：每关给出固定兵符预算（如 **30 点**），武将按稀有度消耗 —— 普通 3 / 精英 6 / 传奇 10。预算用尽则不可再部署新单位。
- **粮草预算**：每关固定粮草（**150 点**，v2），主动技能按技能消耗（见 §3.3），粮草耗尽技能不可放（被动/光环/怒气大招不受影响）。

### 2.4 阵型与先后手
- **不强制阵型**，但提供**预设编队槽（Squad Slots）**：开战前编排上场顺序（最多 8 槽），战斗中按序或自由部署。
- **先后手**：进攻方永远是"主动出手"。开战设 **先手侦查期（8s）** —— 可自由旋转/缩放摄像机、标记防御、但不可部署；满足 SDT 自主，并降低开局信息压力（防认知过载）。
- 部署后单位**立即自动行动**（真·实时），但提供**战术慢放（0.5x）/ 战术暂停**（§7.5）作为 PC 端补偿。

### 2.5 可携带武将数量上限（性能 + 平衡护栏）✅ 已审批锁定
- **单场带兵槽位**：**8 名武将**（与编队槽一致）。
- **同场同名武将上限**：**1**（防止单卡 stack 主导策略 —— 杜绝主导策略红线 ⚠️）。
- **实时同时在场单位上限**：**12**（性能护栏；超出则最老单位不可再部署，直到场上数量下降）。
- **v2 说明**：吴用召唤物使用**独立计数器 `summon_cap=4`**，与 `live_unit_cap=12` 解耦（修正 M4），避免惩罚吴用玩家；粮草一次性消耗，不持续。

### 2.6 节奏档位 · 直控 · 战术指令 · 撤兵（v2 核心新增）

> 本节整合 fun-opinion-li「节奏档位」与「直控 1 将」、fun 文档「战旗指令」、review「撤兵」——**化解"APM 越高越爽"误区，让同一套系统承载三种心流通道**。

**节奏档位（Rhythm Tiers，开局可选，匹配 Bartle 多数派）**
| 档位 | 操作强度 | 玩法 | 适合 |
|------|----------|------|------|
| **导演档** | 轻 | 偏布阵/读图/宏观；发全局战术指令；单位全自动 | 策略党 / Explorer |
| **指挥官档** | 中 | 直控 1 名先锋 + 少量指令 + 怒气交换 | 主流体验 |
| **猛将档** | 高 | 直控 + 全指令 + 快节奏 + 全技能微操 | 动作 / Killer |

**直控层（指挥官/猛将档）**：每局玩家亲自微操 1 名"先锋"武将（WASD/点移 + 手动技能），其余按自动 AI。先锋"重伤退场"则控制权自动转交下一名存活将。交付"兄弟们跟我上"体感，且 8 将上限不变。

**战术指令（导演/指挥官档，覆盖单位 AI 而非取代）**：集火(F) / 后撤(G) / 冲锋(H) / 坚守 —— 短冷却 8–15s、耗粮草 5–10，制造"后撤保血"等微操。决策密度从 ~28 次/局提升至 ~50+ 次。

**撤兵机制（Retreat，落地下 P0-4）**：
- **触发**：选中在场单位 → 点"撤兵"按钮或快捷键 `X`。
- **效果**：单位立即离场；兵符**不返还**（已消耗为沉没成本）；计入 `alive`（否则"撤兵换星"逻辑不成立）。
- **再部署冷却**：撤兵单位进入 20s 再部署冷却（与"重伤退场"同处理）。
- **边缘**：撤兵不可撤销；若单位正处于嘲讽/光环中，效果立即失效。
- **战术暂停上限（P1-11）**：暂停次数上限 3 次（每次 ≤15s）或暂停总时长 ≤60s；0.5x 慢放总时长 ≤30s（防实时紧张感被无限思考瓦解，U1）。

### 3.3 Combo 设计原则（P2 支柱落地 · v2 修订）
- 技能冷却错峰设计（10–22s 不等），鼓励"链"而非"同时全开"。
- **粮草通用约束（v2）**：所有主动技耗粮草 5–30，粮草成为每局通用战术资源（修正 N2）。
- **impact 曲线（v2，采纳 N4）**：`impact_scale = k × sqrt(cooldown_s)`（平方根曲线，比原反比更平缓，避免长冷却技能过强）。
- **弱化预设命名（采纳 emergence §3.1）**：文档保留状态组件自由 combo 教学，但降低"鲁智深→公孙胜"等具名组合的引导性——教程教"状态标签如何交互"而非"用 A 配合 B"，避免把玩家从"发现者"变成"执行者"。
- ⚠️ **防数值膨胀**：技能 impact 受粮草预算（§2.3）与怒气（§3.6）双重约束，避免后期技能碾压。

### 3.4 武魂系统（v2 采纳 · 替代"扩武将池"，解决 combo 空间）

> **取舍依据**：拒绝 review S1「扩池到 12–15」（玩家记不住、广度治标）；采纳 my-takeaway 意见 3——用**武魂附身**在不增加 roster 记忆负担的前提下，把组合空间从 1 种（8 选 8）扩到 **8 × C(30,2) ≈ 3480 种**。

- **机制**：每关开局，玩家从"武魂池"（约 30 个水浒人物，每个仅 1 被动 + 1 小技能，不带本体）中选 **2 个**作为"副将"附身到主将身上。
- **示例武魂池**：花和尚（周围友军受击 -10% / 嘲讽 1.5s）、豹子头（破甲 +10% / 短距突进）、及时雨（召唤上限 +1 / 单体治疗）、智多星（友军攻速 +5% / 召唤 1 援军）、小李广（远程 +1 / 单体狙杀）、浪里白条（水域 +30% / 水域突进）、神行太保（移速 +15% / 短距位移）、鼓上蚤（陷阱不触发 / 短距潜行）……
- **组合空间**：8 主将 × C(30,2) = **3480 种武魂组合**，且**不需新增 8 个完整武将**（仅 1 个 CSV `hero_souls.csv` + 1 个附身逻辑模块）。
- **叙事价值**："武魂附身"= 108 将虽不能都来前线，但"魂"在守护你——**"聚义"主题的直接游戏化**。
- **与技能变体关系**：武魂 = 玩家跨局 deliberate build 选择；技能变体（§3.7）= 局内随机惊喜。两者互补，不冗余。

### 3.5 关系值 + 羁绊技能（v2 采纳 · 轻量版，替代纯预设 combo）

> **取舍依据**：采纳 my-takeaway 意见 2 的"关系 > 配合"方向，但**轻量化**——不推翻状态组件自由 combo，仅叠加"关系值→羁绊"叙事触发层，避免工程过度膨胀。

- **关系定义**（8 将之间）：鲁智深↔林冲（结拜，互救援触发率 +15%）、吴用↔公孙胜（智囊+术士，合击伤害 +10%）、李逵↔鲁智深（莽汉+大哥，并排时李逵移速 +20%）、时迁↔燕青（搭档，潜行时移速共享）。
- **关系值累积**：战斗中通过"互相救援/一起战斗/并排站位"累积。
- **羁绊技能（叙事触发的戏剧时刻，非策划预设 combo）**：鲁智深救林冲（林冲 HP<20% 自动嘲讽+位移救援）、公孙胜+吴用合击（两人皆活且相距<4 格 → 公孙胜 AOE +20%）、李逵护鲁智深（附近 3 格鲁智深受击 → 李逵 5s 攻速 +50%）。
- **好处**：解决"8 选 8 组合空间窄"+"武将没化学反应"+"聚义"落地为机制层。

### 3.6 怒气系统（v2 采纳 · 填补冷却等待真空）

> **取舍依据**：采纳 my-takeaway 意见 6——解决 fun 文档指出的"部署完看戏/等冷却"真空（B5）。是"兄弟叙事"的微操层，成本低。

- **怒气机制**：普攻/受击积攒"怒气"（每武将独立怒气条）。
- **绝技释放**：怒气满可放"绝技"（大招，伤害高 + 视觉强 + **无视冷却**，耗粮草仍计），全屏特写 + 念白。
- **怒气交换（兄弟叙事）**：怒气可"借"给队友——李逵满怒气→点鲁智深头像→鲁智深"罗汉金身"提前释放（李逵怒气清空、鲁智深冷却重置）；借出后借出者 5s"虚弱"（移速 -50%，但下 1 次普攻伤害 +100%，"豁出去"）。
- **好处**：等待冷却期有"积攒/交换怒气"微操；创造"我做了一个取舍"的情感时刻；与直控层天然契合。

### 3.7 技能变体（战场专长，v2 采纳 · 轻量 spice）

> 每局开战，每名上场武将从 3–4 个"战场专长"池随机带 1 个（如林冲：突进留火墙 / 破甲 -60% / 距离 +2）。`hero_skills.csv` 加 `variant_pool` 字段。8 将 × 4 专长 → 组合空间万级，玩家会拼出设计师没预见的 combo（与武魂互补：变体=随机惊喜，武魂=玩家选择）。

## 4. 防守方基地结构（以祝家庄为例 · v2 动态化）

### 4.1 结构层级
| 层级 | 名称 | 性质 | 作用 |
|------|------|------|------|
| 外围 | **外墙（Outer Wall）** | 木质栅栏，可破坏阻挡物 | 第一道屏障，迫使绕行/破墙 |
| 内圈 | **内墙（Inner Wall）** | 石墙，更硬 | 保护核心，构成纵深 |
| 火力 | **箭塔 / 瞭望塔 / 法术塔** | 防御建筑（远程） | 交叉火力，优先索敌 |
| 隐蔽 | **陷阱（陷坑/绊马索/火油）** | 触发式区域 | 埋于预期进攻路径 |
| 驻军 | **庄丁守军（6 兵种，v2）** | 触发式 + AI 分化 | 被进攻触发后出动 |
| 头领 | **Boss 守将（v2）** | 具名反派，守核心 | 击败=胜利前置（§4.9） |
| 核心 | **核心建筑（忠义堂/粮仓/马厩）** | 胜负判定 | 被毁 = 进攻方胜 |

### 4.2 布局原则
- **同心圆 / 分区布局**：核心居中或深藏；塔沿环线布防形成交叉火力；陷阱埋于预期进攻路径。
- **模板 + 结构随机（v2 升级，采纳 emergence §3.2）**：不仅偏移坐标，更随机选择"这面墙木/石""这个塔位是箭塔/瞭望塔"——避免"随机偏移纯 cosmetic"。
- **资源建筑（粮仓/马厩）散布外围**，诱导"为劫掠而分兵"的风险回报（P4 战场体现，且 v2 劫掠为实时动词，§5.2）。

### 4.3 墙的阻挡与破墙逻辑
- 墙 = 可破坏障碍物，有独立 HP。
- 单位默认 AI：**绕墙寻路**，找最近的门/缺口进入。
- 触发破墙：目标被墙阻挡且最近门过远 → 单位转入拆墙模式；破墙专精单位/技能（李逵板斧旋风 ×2、林冲破甲）优先破墙。
- **墙不阻挡远程**：塔、花荣、燕青可从墙外射击墙内 → "远程点塔 vs 近战破墙"核心博弈（P3 落地）。
- **v2 流场分块重算（采纳 H1）**：墙被完全拆毁时只重算墙所在分块及其下游分块（Dijkstra 流场 + 增量更新），避免全图重算卡顿（GDD-03 §⑦）。

### 4.4 塔的射程与火力（基准值 v2，采纳 N3）
| 塔 | 射程（格） | DPS（基准） | 索敌优先级 | 备注 |
|----|-----------|------------|-----------|------|
| 箭塔 Arrow Tower | 5 | **15** | 最近敌方单位 | 主力火力（鲁智深 HP 300 需 20s 击杀，肉盾有意义） |
| 瞭望塔 Watchtower | 7 | **8** | 最近 + 标记/减速 | 控场，可减速敌方 |
| 法术塔 Spell Tower（L4+） | 6 | **5** | 反制召唤/增益 | **驱散改为周期脉冲（§4.6），非常驻** |

- 受鲁智深嘲讽时，塔优先攻击嘲讽目标。
- 塔自身 HP 有限，可被花荣/燕青远程点掉。

### 4.5 核心被毁 = 战败判定（v2 修订）
- **v2**：核心由 **Boss 守将守卫**。摧毁核心的前置是**击败 Boss 守将**（Boss 倒下后核心护盾消失，方可摧毁）。这把"纯建筑血条"升级为"与具名对手的对决"，修叙事空洞（见 §4.9）。
- 若进攻方在核心被毁前**兵力耗尽且无法再部署**（兵符/粮草/怒气用尽、场上无存活单位）→ 进攻方败。
- 防守方无"主动操作"（真·CoC 异步对抗模型），但**动态防御 AI（§4.6）让守军按玩家行为反应**。

### 4.6 动态防御系统（v2 采纳 · 涌现引擎地基 · "对面有脑子"）

> **取舍依据**：这是 8 份文档共同指向的"病根"——对面无脑则再多系统仍是解谜。作为涌现引擎最高杠杆项优先落地。整合 design-emergence §2 + fun-opinion-li「对面有脑子」。

**警报等级（Alert Level，0–4，只升不降）**
| 等级 | 触发 | 敌方行为 |
|------|------|----------|
| 0 平静 | 开局 | 哨兵巡逻、庄丁待命、塔射程-1、陷阱未激活（"黄金窗口"） |
| 1 可疑 | 哨兵听到脚步 | 哨兵警觉、巡逻队移速+30%、瞭望塔旋转搜索 |
| 2 警戒 | 烽火/报警/任何敌单位被杀 | 庄丁出动、塔满射程、援军分批抵达 |
| 3 激战 | 外墙破/Boss 出场/庄丁伤亡≥30% | Boss 出击、法术塔激活、陷阱全激活 |
| 4 绝望 | 核心 HP<30% | 焦土：庄丁狂暴、部分建筑自燃、Boss 不可打断终极技 |

> 关键：警报**只升不降**，玩家无法"消警报"，只能选"在低等级时做多少事"。确保"全程警报 0 通关"理论可行（某些编队），但正面战斗（3–4 级）有对应收益（劫掠更多/用时更短）。

**哨兵（Sentries）**：固定观察单位，扇形视野 120°/5 格，HP 极低；发现敌→点烽火(2s)→警报+1→巡逻队集结。时迁可**静默暗杀**（不触发烽火）。花荣可超远狙杀。

**巡逻队（Patrol Squads）**：2–3 名庄丁沿随机偏移路线循环；发现敌→**不攻击先跑回报警**（信息节点）。林冲可突进追杀信使。受鲁智深嘲讽影响（拉住防报警）。

**敌方指挥官 AI（Commander AI，有限 CP）**：每 15–20s 决策一次，消耗"指挥点(CP)"；基于**它知道的信息**（受战争迷雾限制，非上帝视角）；偏好因关卡主将性格而异（史文恭偏好集火远程、高廉偏好驱散……）。行动：调动巡逻队/增援/激活陷阱/塔集火/关闸门/放法术。

**主动欺骗（Deceive，v2 采纳 fun-opinion-li 2.4）**：玩家可放"诱饵"（草人）引塔/守军偏离防线；或"佯退"引守军追击再回马枪。**前提**是守军 AI 真的会追诱饵/离防线（依赖本节动态防御）。

**庄丁兵种分化（v2 采纳，6 种）**：枪兵（反突进）/ 弓手（点杀低血施法者）/ 哨探（逃跑报信）/ 力士（守核心不追）/ 火攻手（投火油制造燃烧地面）/ 医师（治 Boss）。与英雄形成石头剪刀布涌现（非硬编码克制，AI 行为自然产生）。

**士气系统（v2 暂缓 Phase 3）**：敌方单位士气归零则溃逃；李逵恐惧光环/吴用攻心策反可触发"不战而屈人之兵"。依赖兵种分化后做。

### 4.7 信息战（战争迷雾 + 侦查，v2 采纳 fun-opinion-li 2.3）

> 修"全透明杀死悬念"，把 Explorer 动词延伸到整局。

- **局部战争迷雾**：战场初始被雾覆盖，仅己方单位视野（4–6 格）可见；侦查期 8s 全图短暂可见，结束后迷雾恢复；**侦查期标记永久保存至战斗结束**（采纳 M6）。
- **迷雾 × 英雄**：时迁高视野+潜行=天然侦察兵（"先放时迁探路→调整后续部署"）；花荣超远需前线提供视野→"前哨+狙击"涌现协同；燕青高机动可斥候。
- **实现**：逻辑层 `visibility_system` 维护 `visible_tiles`；表现层 shader 渲染；哨兵/巡逻队脚步声方向性音频（只听声不见人）。

### 4.8 环境系统（v2 采纳 Phase 2 · 最低成本最高涌现）

> 整合 design-emergence §3 + fun 方向 1。题材天然契合（风雪山神庙/火烧草料场/夜袭曾头市）。

- **天气（开局随机/特定关固定）**：晴（基准）/ 雨（火系无效、移速-10%、视野-1、公孙胜雷法×1.5 导电）/ 雾（视野-3、远程-2、时迁更难被发现）/ 风（顺风移速+20%、火随风蔓延）/ 雪（移速-20%、单位留足迹暴露潜行）。
- **昼夜（简化为 toggle，不与天气叠加）**：黑夜视野-4，李逵/时迁/燕青带"夜战"标签不受影响。
- **可互动物（每局随机 2–3）**：火药桶（远程引爆范围伤+破墙）、酒坛（击碎减速/点燃）、粮草堆（劫掠 or 点燃造火墙）、滚石/檑木（合力推下碾压）、信号烽火台（占领后放假信号骗援军）、吊桥（破坏断路径）。**每个物体至少 2 种互斥用法**，制造选择而非正确答案。
- **环境连锁**：火药桶→引燃粮草堆→火墙挡援军；雨天→火系失效→火药桶变障碍。系统间自动产生，不需脚本。

### 4.9 Boss 守将（v2 采纳 · 让"对面有脸"）

> **取舍依据**：采纳 fun 文档「Boss 守将」修叙事空洞 + my-takeaway「反派有脸」+ fun-opinion-li「对面有脑子」；不把"三幕"当第一优先（见 §0.2.2）。

| 关卡 | Boss 守将 | 战斗特点（与英雄机制呼应） |
|------|-----------|------------------------------|
| L1 祝家庄 | 祝氏三兄弟（依次登场，教学） | 逐个击破，难度低 |
| L2 曾头市 | 史文恭 | 高机动、专杀远程 → 需鲁智深嘲讽吸火 |
| L3 大名府 | 梁中书 + 护卫 | 大量召唤杂兵 → 需公孙胜 AOE |
| L4 高唐州 | 高廉（术士） | 驱散脉冲 + 黑风 → 需时迁打断/骗驱散 |
| L5 蔡京府 | 蔡京 + 禁军统领（双 Boss） | 综合考验，核心多层护盾 |

- Boss 守将同时是**指挥官 AI 载体**（§4.6）：它的性格决定指挥偏好。
- 击败 Boss → 核心护盾消失 → 可摧毁核心获胜。
- **招降（v2 采纳）**：击败 Boss 后可消耗声望 + 拨济 +10% 将其招降为可用武将（水浒"招降纳叛"本色）。

## 5. 胜负与评分（v2 修正口径）

### 5.1 三星评分（类 CoC，条件"或"化防认知过载）✅ 已审批锁定
| 星 | 条件 | 说明 |
|----|------|------|
| ★ | 击败 Boss 守将并摧毁核心建筑 | 胜利前提（v2：Boss 倒下后核心可毁） |
| ★ | 摧毁 ≥50% 建筑（含资源建筑，口径见 §5.2 M7） | 鼓励全面劫掠 |
| ★ | **剩余兵力 ≥30% 或 用时 ≤120s**（二选一） | "或"降低双条件卡死 |

> ⚠️ 认知过载红线：第三星用"或"而非"且"。**M5 处理**：保留 L4 锁定的 20–30% 拨济区间（不改动数值），仅在 UI 明确标注"剩余兵力 30%（Star3）"与"拨济 30%（满额上限）"语义不同，化解混淆，不推翻锁定决策。

### 5.2 劫掠收益（v2：P4 动词化）
- **v2 实时动词（采纳 fun-opinion-li 2.2）**：战斗中，任何单位可临时转"搬运"状态（停下战斗扛走资源点，脆弱且脱战）；或地图散布**可即时拾取战利品箱**（谁路过谁捡，但暴露位置引来守军）。把 P4 从结算数字变实时两难。
- **收益类型**：银两（兵符补给/武将升级）/ 粮草（战斗消耗补给）/ 装备（概率掉落）。
- **收益公式（建议，最终数值入 CSV）**：`收益 = 基础值(关卡) × 摧毁比例系数 × 星级系数(1.0/1.3/1.6) × 难度系数`。
- **Star2 建筑口径（采纳 M7）**：`建筑 = 墙 + 塔 + 资源建筑 + 核心`（不含陷阱、不含庄丁）；`buildings.csv` 加 `counts_for_star2` 布尔字段。

### 5.3 劫富济贫的题材闭环（Loot-and-Give，P4 支柱）✅ 已审批锁定
- 劫掠所得中，**20–30% 可选拨济贫民**（L4 锁定），另设"满额拨济额外声望奖励"。
- 拨济 → 提升江湖声望 → 解锁剧情章节 / 新武将 / 新关卡 / **兵书**（v2）。
- **v2 叙事分支（采纳 fun-opinion-li 5 / my-takeaway 5）**：拨济累积比例触发不同剧情线——
  - 高（≥25%）：「民心所向」线 → 解锁义民援军、特殊章节"替天行道"；
  - 中（15–25%）：「亦侠亦盗」线 → 中性结局；
  - 低（≤15%）：「武力威慑」线 → 解锁震慑光环（守军士气-20%）、章节"称霸江湖"。
  - 把拨济从"数值选择"变"身份表达"（SDT 自主）。
- **v2 措辞修正（采纳 T1）**：原"不救济=无惩罚（纯选择）"改为"不救济=选择短期成长，放弃 Meta 进度"，明示取舍；或增加"通关固定少量声望"副路径让拨济是加速器而非唯一通道。
- ⚠️ 支柱漂移/经济失衡防范（补偿设计，L4 锁定）：满额额外声望奖励（`R_bonus = k × 关卡声望基数`）+ 解锁门槛 0.85× 缩放；具体系数待 Phase 3 平衡脚本标定，最终以 CSV 为准。

### 5.4 失败与长尾（v2 新增）
- **超时/投降（采纳 G2）**：超时 = 240s 未毁核心 → 0 星；投降按钮常驻 HUD 右上角，二次确认。
- **残局复盘（采纳 fun 方向 7）**：失败显示"破局点分析"（差 1 格/差 5 秒/差 1 技能），把失败变学习。
- **破釜沉舟（采纳）**：核心被毁前最后 10s 可激活全员狂暴（DPS+50%、HP 持续衰减），创造"绝地反击"传奇时刻。
- **被俘→营救（采纳 fun-opinion-li 2.7）**：特定武将"被俘"（非重伤）时开潜入营救小关（用 stealth 英雄救兄弟）；救回=复仇 buff；失败=招安（5 关后可招降）。
- **Meta 存档（采纳 G5）**：`MetaState`（声望/解锁/兵书/武魂图鉴）由 `src/ui/` 或 `src/main.js` 负责 IO（localStorage/IndexedDB），`src/core/meta.js` 仅状态管理（呼应 AGENTS.md core 不碰 localStorage 红线）。

### 8.3 Phase 3 平衡工具规格（落地 G6 / P2-6 / R1 / demo B3）

> 目的：在数值进 CSV 前，用**离线 AI 托管模拟**标定技能/单位/建筑数值，替代"拍脑袋基准值"。对应 R1 风险（实时战斗数值平衡）与 demo-plan §5.3 B3（8 将技能待逐一平衡）。

**定位**：`tools/raid_sim.js`，无头驱动 `src/raid/core/manager.js`（`RealTimeBattleManager` 逻辑层，node 可直接跑，无 three 依赖）。**与 `tools/sim.js` 区分**：sim.js 跑的是 SRPG 回合制核心（`src/core/BattleManager`），本工具跑劫寨实时模式。

**输入**
- 数据源：`src/raid/core/data.js`（HEROES / BUILDINGS / ENEMIES / LEVEL / SKILL_FX / SCORING，等同于 CSV 唯一数据源）。
- 随机源：`RandomRollSource(seed)`（可复现）或 `FixedRollSource`（回归）；种子经 `--seed` 注入。
- AI 策略脚本：托管决策（部署顺序/技能选点/号令选择/诱饵时机），作为**可替换策略模块**，便于对比不同打法下的数值表现。

**可调参数（CLI / 配置）**
| 参数 | 说明 | 默认 |
|------|------|------|
| `--runs N` | 每配置模拟场数 | 20 |
| `--seed S` | 起始种子（每场递增） | 1000 |
| `--squad a,b,c,d` | 出战编队（heroId 列表） | 4 将标准局 |
| `--weather id\|random` | 指定/随机天气 | random |
| `--order id\|random\|none` | 梁山号令 | random |
| `--knobs k=v,...` | 数值倍率（hp/dps/cost/lc/cd/aoeDmg…） | 1.0 |
| `--json` | 输出 JSON（供回流/CI） | off |

**输出指标（聚合 N 场）**
1. **胜率**（win%）——以少胜多是否成立的核心。
2. **星级分布**（1★/2★/3★ 占比）——检验"或"化星级梯度是否合理。
3. **平均用时 / 超时率**——节奏是否过拖（配合 B1）。
4. **常用 combo 频率**——按英雄技能施放统计 + 3s 窗口内相邻共现对，检验组合空间是否被使用、有无"唯一最优解"。
5. 失败原因分布（wiped / timeout）。

**标定闭环（CSV 回流）**
```
跑 N 场 → 聚合指标 → 比对目标带 → 偏差项给 CSV 修正建议（人工审）→ 改 data.js/CSV → 重跑回归
```
**L1 目标带（验收基线）**：4 将标准局胜率 **65–80%**、平均用时 **60–100s**、超时率 **<10%**、星级峰值落在 **2–3★**、无单一 combo 占比 **>60%**。偏离即触发对应数值行修正。

**红线**：① 随机判定必须经 RollSource 注入，禁止 Math.random；② 数值改动只落 `data.js`/CSV，逻辑层不硬编码；③ 模拟结论须人工复核后才回流，不自动改表。
