# 劫寨战斗 Demo 实施计划（Three.js · 可体验版）

> 日期：2026-07-21
> 目标：在浏览器里**亲手玩一局**劫寨战斗，体验「部署→实时攻防→Boss 对决→劫掠拨济」的核心手感，验证设计 v2 的"修循环"是否成立。
> 上游：`combat-mode-design.md`（v2）、GDD-01~04。
> 范围原则：**Demo 不是完整游戏**——从 v2 全量中裁剪出"最能证明循环好玩"的最小集合，其余标注为后续迭代。

## 3. 技术方案

### 3.1 架构：逻辑 / 表现分离（沿用工程红线）

```
src/
  raid/                    # 劫寨 Demo 独立模块（不污染 SRPG）
    core/                  # 纯逻辑，node 可测
      realtime_battle_manager.js   # 新建：替代 SRPG BattleManager
      flow_field.js                # 新建：流场寻路（绕墙/破墙）
      deploy_system.js             # 部署许可/兵符/冷却/撤兵
      alert_system.js              # 警报等级/哨兵/巡逻
      unit_ai.js                   # 实时索敌/攻击/移动
      hero_skills.js               # 8 将技能（主动/被动）
      rage_system.js               # 怒气
      loot_system.js               # 劫掠搬运
      scoring.js                   # 三星/拨济
      data/raid_l1.csv             # L1 布局 + 武将/塔/庄丁数值
    render/                # Three.js 表现
      raid_scene.js                # 场景/棋盘/建筑/单位网格
      unit_mesh.js                 # 单位模型（先用色块+头顶血条）
      fx.js                        # 破墙粒子/塔火弹道/技能特效/2-3 个高光时刻
      camera_ctrl.js               # 旋转/缩放/平移
    ui/
      raid_hud.js                  # 编队栏/资源条/技能栏/怒气条/拨济结算
      raid_input.js                # 键鼠/直控/指令/暂停慢放
    main_raid.js           # Demo 入口（替代/并存 src/main.js）
```

**复用现有 SRPG core（不重写）**：`grid.js`、`unit.js`、`damage_calculator.js`、`effect_system.js`（状态组件）、`targeting.js`、`roll_source.js`、`csv.js`、`data_loader.js`。
**新建**：RealTimeBattleManager（20Hz 固定步长 + 插值）、流场寻路、动态防御 AI、部署/劫掠/怒气/评分。
**不改动** SRPG 现有文件（架构红线）。

### 3.2 关键技术决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 寻路 | **流场（Flow Field）**，墙破时分块重算 | 12+ 单位实时，比逐单位 A* 省性能（R2） |
| 逻辑步长 | **20Hz 固定步长** + 表现层插值 | 可测、可回放、与表现解耦 |
| 随机 | `RollSource` 注入（哨兵警觉/庄丁增援/闪避） | 架构红线，可设种子复现 |
| 状态组件 | 复用 `effect_system.js`，嘲讽/减伤/潜行/破甲以组件叠加 | Combo 与驱散脉冲的基础 |
| 胜负判定 | 逻辑层广播 `battle_end_event`，UI 一次性结算 | 与 GDD-04 口径一致 |
| 数值来源 | 全部入 `data/raid_l1.csv`，无硬编码 | 沿用"CSV 唯一数据源"红线 |

## 4. 里程碑（建议 4 个迭代，每迭代可玩）

### 迭代 1 · 跑通循环骨架（逻辑优先）
- 场景：棋盘网格 + 固定 L1 布局（墙/塔/陷阱/核心/Boss 占位）。
- 部署：编队栏 + 兵符/冷却校验 + 侦查期 8s。
- 实时循环：流场寻路 + 单位自动索敌/攻击 + 技能手动触发（先做鲁智深嘲讽、公孙胜 AOE、花荣狙击 3 个代表）。
- 胜负：毁核心→三星结算页（数字）。
- **验收**：能从部署打到结算，无迷雾/无动态防御/无直控。

### 迭代 2 · 对面有脑子（动态防御）
- 警报等级 0–3；哨兵扇形视野点火；巡逻队报警；庄丁增援被破墙方向。
- Boss 守将：警报 3 或核心被围时出击，击败后核心可毁。
- **验收**：玩家能感受到"守军在反应我"（哨兵点火→巡逻来→庄丁增援破墙口）。

### 迭代 3 · 手不停 + 水浒魂（输入与反馈）
- 直控 1 先锋（WASD/点移）+ 4 战术指令。
- 怒气系统（满怒大招）+ 劫掠搬运（粮仓）。
- 2–3 个高光时刻（林冲突进特写 / 花荣狙杀震屏 / 核心摧毁慢动作）。
- 拨济滑条结算。
- **验收**：120s 内有连续操作；击败 Boss 有对决感；劫掠/拨济即时可见。

### 迭代 4 · 打磨与验证
- 补全 8 将技能；战术暂停/慢放；摄像机手感；性能（12 单位 60fps）。
- 体验自测 Q1/Q2/Q3，记录问题，回流设计文档修订。

### 5.3 待平衡/后续迭代项（自测记录）

| # | 项 | 现象 | 建议 |
|---|----|------|------|
| B1 | 3 将低速局可能超时 | 3 将不劫掠直推时 120s 未分胜负 | 教学关建议 4 将起手；或核心 HP 再降 / 增援再少；后续用平衡脚本标定 |
| B2 | 劫掠引导弱 | 玩家可能不打粮仓直接推核心 | 粮仓加"高亮+靠近提示"；拨济结算强化收益感 |
| B3 | 技能数值待标定 | 8 将技能仅用基准值，未逐一平衡 | ✅ **已闭环**（2026-07-23）：`tools/raid_sim.js`（规格 §8.3）AI 托管跑 N 场输出胜率/星级/用时/combo。基线 100% 胜/18.4s → 经 `--sweep` 扫描定位难度杠杆在**守军存活(ehp)×守军输出(edps)+BOSS**（核心 HP/塔非瓶颈；集中 BOSS 威胁拐点太陡）。已回流 `data.js`：庄丁 360/15.75、枪兵 560/20.25、哨兵 80、BOSS祝龙 1610/32.5 → **无旋钮实测 76% 胜 / 72.4s / 超时 0%**，全部落带（65–80% 胜 / 60–100s），21 测试全绿。**L2 曾头市已接入**（2026-07-23）：`data.js` 改 `LEVELS`+`getLevel`、manager 支持 `levelId`、raid_sim 加 `--level`/`--curve`；难度曲线 L1 74–76% / L2 66% 双落带（L2 带 50–75%）。L2 用专属铁骑(420/16 肉度+低攻)+更强史文恭(2000/40)+与 L1 同款安全塔位，难度来自**小兵墙+BOSS**而非堆塔（见下方「难度设计教训」）|
| B4 | 音效缺失 | 高光时刻仅视觉，"燃"不足 | 迭代补关键音效（破墙/Boss/三星/绝技） |

> **难度设计教训（L2 标定沉淀，供 L3+ 复用）**
> 1. **堆敌人/堆塔不会变难**：因"击杀回血 8%"机制，更多可击杀目标 = 更多免费回血 = 反而更易。L2 初版（多 2 塔 + 脆皮铁骑）曾 83% 胜，比 L1 还易。
> 2. **单一 BOSS 当门神会二元化**：BOSS 血量略升即胜率从 85% 直跌 0%（秒 BOSS 赢 / 被秒团灭），无平滑带。落带靠**「坦克小兵提供持续分布式压力 + 中等 BOSS」**组合（L1 庄丁 360 血/15.75 攻即此范式）。
> 3. **小兵要"高血低攻"**：高攻小兵 = 送死团灭；低攻肉盾 = 扛住给持续压力又不秒人。L2 铁骑定为 420 血/16 攻。
> 4. **塔位避开唯一大门**：贴门塔形成"绞肉机"关卡，配合肉度小兵直接 0% 胜。L2 只用与 L1 同款外翼安全塔位。
> 5. **实用旋钮**：`--sweep` 扫 `ehp×edps×bossHp` 找落带；整关调难度优先动"守军量/小兵墙"（对 AI 胜率最敏感），BOSS 旋钮在临界处失灵。

## 5.4 迭代 5 · 涌现深化（治背板）—— 已落地 ✅（2026-07-21）

> 目标：让"每局不一样"，从"可背板的静态谜题"变"有变量的动态博弈"。浏览器实测 + 4 天气全通关验证。

| 机制 | 实现 | 涌现点 |
|------|------|--------|
| **战争迷雾** | 单位视野（近战4/远程5/时迁6），迷雾外敌人/建筑不渲染；侦查期全图可见+标记持久；雾中敌人真实行动 | 探索动词进战斗；时迁从"切核心"扩为"侦察核心" |
| **天气环境** | 开局随机晴/雨/雾/雪：雨=火系无效+雷法×1.5+移速-10%；雾=视野-3+远程-2+潜行增强；雪=移速-20%+足迹暴露潜行 | 同一关不同天气编队/战术价值重排（治背板核心） |
| **主动欺骗（诱饵）** | 粮草 15 放草人，吸引范围 5 内塔/守军优先攻击 8s | P1"智取"可操作化：用 1 草人换半边火力 |
| **梁山号令** | 战前三选一：粮草先行(+50粮)/火攻计(火系+50%)/夜行衣(部署4s潜行) | build 多样性：同一关不同号令战术倾向不同 |

**验证**：21/21 单元测试全绿（新增迷雾/天气×3/诱饵/号令×3 用例）；4 种天气下 4 将标准局均 20–22s 通关（雪天减速略慢、雨天雷法放大，均未破坏平衡）；浏览器实测号令界面/迷雾渲染/天气粒子/诱饵扣粮草全部生效，零报错。

**待后续**：庄丁 6 兵种分化、可互动物（火药桶/烽火台/吊桥）、关系羁绊、音效。
